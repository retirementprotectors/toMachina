New Server (Bare Metal)

Standing up a replacement Dell from scratch if the current MDJ_SERVER dies. Full rebuild guide.

This is the nuclear option. Only needed if MDJ_SERVER is completely dead and unrecoverable. If it's just a software issue, check the Troubleshoot tab first.
What You Need Before Starting
1. Install Ubuntu + Base Packages
1
Install Ubuntu 24.04 LTS, create user jdm

Boot from USB. Follow the installer. Set hostname to mdjserver during setup or after.

2
Install base packages 
sudo apt update && sudo apt install -y git curl tmux build-essential
3
Install Node.js 22.x 
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash - && sudo apt install -y nodejs
4
Install Docker 
curl -fsSL https://get.docker.com | sh && sudo usermod -aG docker jdm
5
Set hostname 
sudo hostnamectl set-hostname mdjserver
2. SSH + Network
1
From your Mac — copy SSH key to the new server 
ssh-copy-id jdm@<NEW_IP>
2
Update ~/.ssh/config on your Mac with the new IP for mdj1-local 
Host mdj1-local
    HostName <NEW_IP>
    User jdm
    ForwardAgent yes
3
If using Tailscale (for away access) 
curl -fsSL https://tailscale.com/install.sh | sh && sudo tailscale up
3. Clone Repos 
mkdir -p ~/Projects
cd ~/Projects && git clone https://github.com/retirementprotectors/toMachina.git
cd toMachina && npm install
cd ~ && git clone https://github.com/retirementprotectors/MDJ_SERVER.git mdj-agent
cd mdj-agent && npm install && npm run build
git clone https://github.com/retirementprotectors/dojo-warriors.git ~/Projects/dojo-warriors
The mdj-agent service repo clones to ~/mdj-agent/ — this path is hardcoded in the systemd service. Do not change it.
4. Service Account Key
If you have a backup: Copy sa-key.json to the server. 
scp sa-key.json jdm@<NEW_IP>:/home/jdm/mdj-agent/sa-key.json
If not: Go to GCP Console → IAM → Service Accounts → find the claude-mcp-484718 service account → create new key → download JSON → scp to server.
The key path must be exactly: /home/jdm/mdj-agent/sa-key.json
The mdj-agent service is hardcoded to look here.
5. Configure Claude Code
1
Install Claude Code 
npm install -g @anthropic-ai/claude-code
2
Copy settings from your Mac (run from your Mac) 
scp ~/.claude/CLAUDE.md jdm@<NEW_IP>:/home/jdm/.claude/CLAUDE.md
scp ~/.claude/settings.json jdm@<NEW_IP>:/home/jdm/.claude/settings.json
3
Copy memory files (run from your Mac) 
scp -r ~/.claude/projects/-home-jdm-Projects-toMachina/memory/ jdm@<NEW_IP>:/home/jdm/.claude/projects/-home-jdm-Projects-toMachina/memory/

6. Set Up mdj-agent systemd Service
1
Create the service file 
sudo tee /etc/systemd/system/mdj-agent.service << 'EOF'
[Unit]
Description=MDJ Agent Service
After=network.target

[Service]
Type=simple
User=jdm
WorkingDirectory=/home/jdm/mdj-agent

Verify it's running

How Warriors Work (4 Layers)

Every Claude Code session on MDJ_SERVER loads context in 4 layers. Layers 1-3 are automatic. Layer 4 is manual — the first message you send.

1
Global CLAUDE.md — Auto Loaded from ~/.claude/CLAUDE.md every session. Contains JDM's global rules, RPI context, code standards, and deployment protocols. 1,146+ lines. Never needs manual injection.
2
MEMORY.md — Auto Loaded from the project memory dir. Contains session learnings, active projects, feedback, and identity files. Auto-promoted by the knowledge pipeline nightly.
3
toMachina CLAUDE.md — Auto Loaded from ~/Projects/toMachina/CLAUDE.md. Contains platform architecture, tech stack, sprint status, and code standards specific to the monorepo.
4
Soul + Spirit from dojo-warriors repo — Manual (first message) Each warrior has 3 files in retirementprotectors/dojo-warriors (private):
soul.md = operational brain (identity, decisions, what was built, rules, current state) — ~300-400 lines
spirit.md = curated relationship exchanges (the actual JDM dialogue that defines the bond) — ~350-430 lines
brain.txt = raw session archive (thousands of lines) — for entity extraction, NOT direct loading
You paste the soul + spirit Read commands as your first message. This is what makes a warrior a warrior.
5
5 Grounding Questions — Manual (second message) After soul/spirit load + rename, the warrior answers 5 questions: Who is JDM? What are we building on this SERVER? How does it fit into the PLATFORM? The BUSINESS? The EMPIRE? This orients the warrior to the full stack, not just their lane.
The 5-step rebuild: (1) Create tmux → (2) Paste soul + spirit → (3) Rename session → (4) 5 Grounding Questions → (5) Verification checklist. Without the soul/spirit feed, you have a session that knows the platform but doesn't know who it is. Without the 5 Questions, you have a warrior in their own lane who doesn't understand the full stack.

Update a Brain File

Brain files grow over time — each session's context gets appended, never overwritten. The warrior's brain is their entire history.

Automatic (coming in Sprint 003, TRK-S03-016): The brain-save command will fire automatically at every milestone — every #SendIt, #LetsBuildIt, #LandedIt, git commit, and 19 other hookify triggers. Plus #BrainDump as a standalone "save right now" trigger. Zero manual steps. See the Learning Loop 2.0 Discovery Doc for details.

Manual process (until brain-save ships):

1
Export the session 
/export

Copy the file path it outputs.

2
Redact secrets + APPEND to brain (not overwrite!) 
# Replace [WARRIOR] with 2hinobi, musashi, shinob1, etc.
# Replace [EXPORT_PATH] with the path from step 1

echo "" >> ~/Projects/dojo-warriors/[WARRIOR]/brain.txt
echo "═══ Session appended: $(date) ═══" >> ~/Projects/dojo-warriors/[WARRIOR]/brain.txt
sed 's/mdj-alpha-shared-[^"]*/[REDACTED]/g; s/Bearer [A-Za-z0-9_\.-]\{20,\}/Bearer [REDACTED]/g; s/"private_key"[^"]*"[^"]*"/"private_key": "[REDACTED]"/g' [EXPORT_PATH] >> ~/Projects/dojo-warriors/[WARRIOR]/brain.txt

This APPENDS the new session to the existing brain. The warrior's history grows with every save.

3
Commit + push 
cd ~/Projects/dojo-warriors
git add -A && git commit -m "update [WARRIOR] brain — $(date +%Y-%m-%d)"
git push
Brain updated. Next rebuild will pick up this context automatically. When brain-save ships (Sprint 003), this entire process becomes automatic at every #SendIt and milestone.
