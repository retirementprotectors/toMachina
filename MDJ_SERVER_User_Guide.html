Connect to MDJ_SERVER

Always use the local IP. One command. Done.

🏠 HOME Network

ssh mdj1-local

Resolves to 192.168.6.93 · Instant · Rock solid

✈️ AWAY Network

ssh mdj1

Requires Tailscale running + connected · Resolves to 100.99.181.57 · Can be flaky — if it fails, use MDJ Mobile instead

🚨 NEVER type ssh mdj1 when you're home — it will just hang.
ssh mdj1 always routes through Tailscale (100.99.181.57) regardless of your network. Tailscale is currently down on MDJ1, so it just sits there doing nothing. Always use ssh mdj1-local at home — it goes straight to the local IP and connects instantly.

New Machine Setup

One-time steps to connect a new Mac to MDJ_SERVER.

1
Copy your SSH key to MDJ1 (enter password once — never again) 
ssh-copy-id jdm@192.168.6.93
2
Add SSH alias to ~/.ssh/config 
cat >> ~/.ssh/config << 'EOF'
Host mdj1-local
    HostName 192.168.6.93
    User jdm
    ForwardAgent yes
EOF
3
Test connection 
ssh mdj1-local

You should see: jdm@mdjserver:~$

Done. Three steps. You're in.

Troubleshooting

Something broke? Start here.

SymptomFixNotes
SSH hangs / no connectionssh jdm@192.168.6.93Use raw IP if alias isn't working. Must be on home network.
Agent service down (connection refused on :4200)ssh mdj1-local "sudo systemctl restart mdj-agent"Wait 10 sec then re-check health.
tmux session gone after reboottmux new -s SESSION_NAMESessions don't survive reboots. Just start a new one.
Need to reboot MDJ1ssh mdj1-local "sudo reboot"Wait 2 min. Agent auto-starts. tmux gone.
Claude Code hangs on startupssh mdj1-local then rm -rf ~/.claude-code && claudeNukes runtime cache only. Config preserved.
ssh mdj1 won't connect (away)Make sure Tailscale shows "Connected" in menu bar, then retry. If still failing — use MDJ Mobile instead.Tailscale SSH is unreliable. MDJ Mobile is the better away option.
Tailscale not running (away)open -a Tailscale — wait for "Connected" in menu barRequired for any away access to MDJ1.
