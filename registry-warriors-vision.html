The Breakthrough

On the night of April 4, 2026, a pattern emerged from months of building. It started with a simple observation:

"The thing is, with the rest of you guys who aren't working in that shit OFTEN, you get out of Context on it. So when we're building an Audit or something — you'll go 'Let me scope this out and build one of these and one of those.' And I have to go — Hang on. CHECK ATLAS FIRST. And then you come back and go 'Holy Shit! This is 80% done already!'"

— JDM

That frustration — warriors reinventing solutions that already exist in ATLAS — revealed something deeper. Some warriors write code. Others compose capabilities. They are fundamentally different kinds of intelligence.

Two Classes of Warrior

The Dojo has always had warriors. But not all warriors fight the same way.

ClassWarriorsHow They WorkOutput
Code WarriorsRAIDEN + RONINWrite code. Branches, PRs, deploys. Fix bugs, build features.Commits on main
Registry WarriorsMEGAZORD + VOLTRON + MUSASHICompose Tools → SuperTools → Wires. Execute through CCSDK sub-meshes.Executed pipelines, processed data, delivered assets

Code Warriors solve problems by changing the platform. Registry Warriors solve problems by using the platform.

RAIDEN and RONIN write the code that builds the Tools. MEGAZORD, VOLTRON, and MUSASHI compose and deploy those Tools to solve business problems. The code warriors build the weapons. The registry warriors wield them.

The Mecha Principle

A Registry Warrior doesn't use their mesh. They become it.

  • VOLTRON forms the Lions — each Lion is a product-type specialist (Medicare, Annuity, Life/Estate, Investment, Legacy/LTC). When a client case needs Medicare expertise, VOLTRON doesn't call the Medicare Lion. VOLTRON becomes the Medicare Lion.
  • MEGAZORD morphs into the Rangers — each Ranger is a data-domain specialist (Import, Commission, ACF, Correspondence, Reference). When data needs importing, MEGAZORD doesn't delegate to the Import Ranger. MEGAZORD becomes the Import Ranger.
  • MUSASHI deploys the Artisans — each Artisan is a channel specialist (Print, Digital, Web, Social, Video). When a brochure needs creating, MUSASHI doesn't outsource to the Print Artisan. MUSASHI becomes the Print Artisan.

This is the Mecha Principle: the leader and the specialists are the same entity in different configurations.

Command Center App Spec

Each Command Center is a cross-portal App (own branding, appears in ProDash/RIIMO/SENTINEL sidebar). Shared layout:

  • Registry View — browsable inventory of Tools, Super Tools, and Wires. Search, filter, status indicators.
  • Pipeline View — ticket kanban with phase toggles (same pattern as The Dojo, but domain-specific stages)
  • Mesh View — CCSDK specialist status, assignment, execution history
  • Health Dashboard — source staleness, wire execution frequency, data accuracy metrics
  • Intake Funnel — domain-specific input channel (how work arrives)
