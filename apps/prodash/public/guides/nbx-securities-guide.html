1 What is NBX Securities?

NBX (NewBizXpress) is how we process new business cases for securities and advisory accounts through Gradient. Every client who needs a Schwab (RIA) or RBC (BD) account comes through this pipeline.

Think of it as a Kanban board — like a visual to-do list where each case (card) moves left to right through 7 stages, from "New" all the way to "Funded."

Why does this matter? When you open a case and see the task checklist, you're looking at Tasks grouped into Steps, inside a Workflow, for the current Stage, of this Pipeline. The grouping headers (DOCUMENT_INTEGRITY, DATA_COMPLETENESS, etc.) are the Steps. Each checkbox is a Task.

3 The 7 Stages

Every case flows through these stages from left to right. Stages with a lock are gated — all required tasks must pass before the case can move forward.

1. New
Case just created. Assign advisor, create Drive folder.
2 tasks • No gate
2. Intake
Gather statements, ID, existing forms. Conduct intro meeting. Complete risk profile.
6 tasks • No gate
3. Suitability lock
Verify client profile (net worth, income, risk tolerance). Product suitability checks. CAF signing + Best Interest rationale.
10 tasks (4 system checks) • Gated
4. QC Review lock
Document integrity, data completeness, compliance checks. Verify correct custodian, no Signal references.
11 tasks (3 system checks) • Gated
5. Ready to Submit lock
Generate DEX form kit, review, send DocuSign. Confirm all docs signed.
5 tasks (2 system checks) • Gated
6. Submitted
Package sent to custodian/carrier. Track confirmation. Resolve any NIGO items.
3 tasks • No gate
7. Funded
Account funded. Notify client. Link account in ProDashX for ongoing management.
3 tasks • No gate

6 Completing Tasks & System Checks

Manual Tasks

check_box

Click the checkbox next to a task

When you've completed the work (e.g., collected the statement, verified the signature), click the checkbox. It turns green with a checkmark and records who completed it and when.

warning
Be honest with checkboxes. These are compliance records. Only check a task when the work is actually done. QC will verify.

7 Understanding Gates

Gates are the guardrails. Three stages have gates: Suitability, QC Review, and Ready to Submit.

When a stage is gated, the system will NOT let you advance the case until every required task (marked with *) is completed. This prevents incomplete or non-compliant cases from moving forward.

What Happens When You Try to Advance Past a Gate

  1. You click Advance Stage
  2. The system checks all required tasks in the current stage
  3. If any required task is still PENDING, you get a red error message listing the blockers
  4. Complete the remaining tasks, then try again
gpp_bad
Example from Clark, Sheila: QC gate was blocked because: net worth missing on CAF, document alteration detected, BI rationale was copy-pasted from Budd case (92% similarity), and Signal Wealth was still referenced in the Products Considered field. All 7 blockers had to be resolved before submission.

9 Moving Cases Between Stages

Method 1: Advance Stage Button (Recommended)

arrow_forward

Open the case, then click "Advance Stage"

This is the safest way — it checks gates and only allows the move if all required tasks are done. The case moves to the next stage in order.

Method 2: Drag and Drop

drag_indicator

Drag a card from one column to another

On the Kanban board, you can drag a card and drop it into any column. This is an admin override — it bypasses gate checks. Use with caution.

warning
Drag-and-drop skips gates. Only use this if you have a specific reason (e.g., Josh instructed you to move a case). For normal workflow, always use the Advance Stage button inside the case detail.

11 FAQ

Q: I accidentally checked a task that isn't actually done. Can I undo it?

Not yet — completed tasks can't be unchecked in the UI. Let Josh or Nikki know and they can reset it from the admin side.

Q: A system check is showing "Pending" but I've already done the work. Why?

System checks verify data in the system, not whether you did the work. If the data hasn't been entered into the client record yet (e.g., net worth on the CAF), the check will stay Pending. Update the client record first, then the check should pass.

Q: What's the difference between the Pipeline Board and Pipeline Studio?

Pipeline Board (under Sales/Service) = where you WORK cases. See cards, complete tasks, advance stages.
Pipeline Studio (under Apps) = where ADMINS configure the pipeline structure. Stages, steps, task templates. You don't need Pipeline Studio for daily work.

Q: Can I see all pipelines, not just NBX Securities?

Yes — there are 14 pipelines total (Annuity, Life Insurance, Medicare, etc.). Each appears in the Sales or Service sidebar sections. NBX Securities is the first one we're actively using with real case data.

Q: What do the priority colors mean?

HIGH = Urgent — large AUM, time-sensitive, or compliance issue
MEDIUM = Normal workflow
LOW = Can wait — smaller accounts, no deadline pressure

Q: How do I change who's assigned to a case?

Open the case detail, click the person badge (shows current assignee email) or click the Reassign button at the bottom. Enter the new person's @retireprotected.com email.
