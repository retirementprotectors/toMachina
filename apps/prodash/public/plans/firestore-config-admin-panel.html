Phase 3 — Field-Level Add / Delete

P32 tickets — Every field can be added or removed
TRK-511BOTHAdd Field — "+ Add Field" button at top of expanded doc. Inline row: [name input] [type select: string/number/boolean/array/object] [Create]. Default values: ""/ 0 / false / [] / {}. Field name validation: snake_case only, no spaces, no duplicates, no leading underscore. Sets dirty → Save persists.
TRK-512BOTHDelete Field — X button on every field label. Confirmation popup. Removes from editData AND tracks in deletedFields[] state. On Save: sends _delete_fields array alongside remaining data — required because plain omission from payload does NOT delete fields in Firestore.
