Close every gap between file intake and Firestore. When done, all 4 intake sources run end-to-end through WIRE_INCOMING_CORRESPONDENCE with zero manual steps.

The Problem

We have 4 working intake sources that queue files to intake_queue. We have a working wire executor that orchestrates 6 super tools. But they don't connect. The wire expects local filesystem image paths. The intake queue provides Drive file IDs. There's no bridge.

Dependencies

  • Phase 2 can be built in parallel with Phase 1 (different files), but can't be tested until Phase 1 works.
  • Phase 3 is independent — can be built anytime.
  • Phase 4 requires Phases 1 + 2.
  • Phase 5 requires Phase 4 (need real wire executions to test approval flow).
  • Phase 6 can start after Phase 1 (backfill script is independent).

Risk

  • PDF→image conversion in Cloud Functions: May not support native dependencies (poppler, ghostscript). Fallback: run wire execution on Cloud Run (tm-api) instead of Cloud Functions. tm-api already has a Dockerfile we control.
  • Claude Vision API cost: Each document extraction = 1 API call. At ~277 clients × ~6 docs average = ~1,660 calls. At ~$0.01/call = ~$17. Not a concern. But don't run on all 1,807 files at once — batch by priority.
