Notifications Module

Portal-wide activity center with 6 tabs. Surfaces all Firestore writes, wire outputs, and approval workflows. Replaces archived RAPID_IMPORT approval UI.

Builder 1: CORE + INFRA

TRK-278: Firestore Collection + Types

  • Create packages/core/src/notifications/types.ts with Notification interface
  • Fields: type, entity_type, entity_id, entity_name, summary, fields_changed, source_type, source_id, source_label, user_id, team_scope, hyperlink, approval_batch_id, read, created_at, portal
  • Add notifications to firestore.rules with isRPIUser() gate
  • Composite indexes: [portal, created_at], [entity_type, created_at], [user_id, read, created_at], [source_type, created_at]

TRK-279: Cloud Function onWrite Triggers

  • Create services/intake/src/notification-triggers.ts
  • onWrite on clients → compare before/after → create notification with changed fields summary
  • onWrite on clients/{id}/accounts → same pattern
  • Generate summary: "Phone updated: (515) 555-1234 → (515) 555-5678"
  • Generate hyperlink: /contacts/{clientId}

TRK-280: createNotification() Utility

  • Create packages/core/src/notifications/create-notification.ts
  • Used by: wire executor (after SUPER_WRITE), approval.ts (on create/execute), Cloud Functions
  • Auto-sets: id, read=false, created_at

Builder 2: UI

TRK-281: Slide-out Shell + Tab Bar

  • Create packages/ui/src/modules/Notifications/NotificationsModule.tsx
  • Slide-out panel: same width/animation as Communications Module
  • Tab bar: ALL / CONTACT / ACCOUNT / MyRPI / DATA / APPROVALS
  • Firestore onSnapshot subscription filtered by portal

TRK-287: APPROVALS Tab (Most Complex)

  • List view: pending approval_batches — source info, client name, field count, source wire
  • Detail view: expand to field-level review — each field editable inline
  • Actions: Approve All / Reject / Request Info / Execute
  • Execute calls POST /api/approval/batches/:id/execute (already built)
  • No new API routes needed — uses existing approval.ts endpoints
  • This is the ProDashX-native replacement for the archived RAPID_IMPORT approval UI

TRK-282–286: Five Filter Tabs

  • ALL: all notifications, reverse-chronological, paginated (50 initial + load more)
  • CONTACT: filter entity_type == 'client'
  • ACCOUNT: filter entity_type == 'account'
  • MyRPI: filter user_id == currentUser or source_id == currentUser.email
  • DATA: filter source_type in ['wire','import','intake']

TRK-288 + TRK-289: Row Component + Unread Badge

  • NotificationRow: type icon + entity name + summary + relative timestamp + unread dot
  • Click → navigates to hyperlink in main content area + marks as read
  • Badge: unread count on sidebar icon via onSnapshot query
  • Bulk "Mark all read" button in slide-out header

Key Notes

  • Real-time: Firestore onSnapshot for live updates. No polling.
  • APPROVALS reads approval_batches directly — not the notifications collection. Separate data source.
  • No new API routes. All data from Firestore + existing approval.ts.
  • Slide-out width: matches Communications Module. Respects real estate constraints.
