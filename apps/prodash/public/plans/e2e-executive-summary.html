Automated Quality Assurance

What It Means For Our Team

Live in Production March 2026

The Problem We Were Facing

  • Every time we shipped an update to ProDashX, RIIMO, or SENTINEL, there was no automated way to verify it actually worked. Someone had to go click through everything manually.
  • We shipped 674 features across 9 sprints with zero automated verification — every one of those was a leap of faith.
  • After every release, the CEO had to manually click through 470+ items to make sure nothing broke. That is not a sustainable or scalable process.
  • Bad data could slip in completely undetected. Real example: 619 client phone numbers from the CoF import were silently corrupted — the last 1-3 digits were wrong on every single one. Nobody caught it until Vinnie noticed on a live call with a client. That is unacceptable.
  • A single typo in code could break the entire platform and we would not know until someone complained.

What We Built

The platform now tests itself before every code change goes live. Here is what that looks like:

  • 12 visual checks run automatically: the system opens a real browser, logs in to ProDash, navigates to every module — Contacts, Accounts, Admin, FORGE, Pipeline Studio, Communications, RPI Connect, MyRPI, the Sidebar, and the Header — and verifies everything renders correctly. If a page is blank or broken, it catches it.
  • 4 pipeline tests verify that document processing works end-to-end: mail intake, specialist uploads, and ACF filing. These are the workflows the Service team depends on every day.
  • Security scanning (Dependabot + CodeQL) runs continuously, monitoring our code and every software dependency for known vulnerabilities. If a security issue is found, we are alerted immediately.
If ANY of these checks fail, the code cannot be deployed to production. Period. The gate is automatic and cannot be overridden.
