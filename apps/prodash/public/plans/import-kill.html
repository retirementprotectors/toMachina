1. Current State

toMachina has 63 Cloud Run API routes, 4 intake Cloud Functions, a full packages/core business logic layer, and 3 deployed portal apps. The GAS migration (Sprint 4-5) replaced 6 engines: RAPID_COMMS, RAPID_FLOW, RAPID_API, C3, ATLAS, CAM. Two remain:

RAPID_IMPORT

30 .gs files · 106,641 lines

Data ingestion engine. Handles Book-of-Business imports, account processing, agent onboarding, commission reconciliation, client enrichment, validation, and carrier seeding. Most functions already replaced by Cloud Run routes + intake Cloud Functions.

RAPID_CORE

17 .gs files · 16,211 lines

Shared GAS library. Provides Sheets adapter (getMATRIX_ID, TABLE_ROUTING), normalization helpers, matching, validation API, carrier lookups, Drive ops, org admin, entitlements. All business logic already ported to packages/core. Only remains as runtime dependency for RAPID_IMPORT.

Key insight: RAPID_CORE only survives because RAPID_IMPORT calls it. Kill RAPID_IMPORT's last 4 functions and RAPID_CORE dies automatically — no consumers remain.

3. The 4 Functions to Port

Only 4 distinct GAS functions block the kill. Two wires share processAccountImport, so there are 4 unique ports covering 5 wires.

FunctionSource FileBlocking WiresWhat It DoesTarget Location
scoreContactQuality IMPORT_Validation.gs WIRE_VALIDATION_PIPELINE Scores contacts 0-100 based on completeness (phone, email, address, DOB). Flags low-quality records for enrichment. packages/core/src/validation/ + expose via services/api/src/routes/clients.ts
processAccountImport IMPORT_Account.gs WIRE_LIFE_ANNUITY_ACCOUNTS, WIRE_INVESTMENT_ACCOUNTS Parses carrier/custodian account files. Maps columns to schema. Dedup against existing accounts. Writes to _ACCOUNT_ANNUITY or _ACCOUNT_INVESTMENTS. services/api/src/routes/import.ts (extend existing import route)
processAgentImport IMPORT_Agent.gs WIRE_AGENT_MANAGEMENT Imports agent/producer records from Gradient. Maps appointment data. Dedup against _AGENT_MASTER. Creates/updates producer records. services/api/src/routes/import.ts (extend existing import route)
processCarrierSeed IMPORT_Client.gs or dedicated WIRE_NAIC_CARRIER_SEEDING Seeds carrier collection from NAIC data export. Parses NAIC codes, company names, state domicile, lines of business. services/api/src/routes/carriers.ts (extend existing carriers route)

All 4 functions follow the same pattern: parse input file → normalize fields (using RAPID_CORE helpers) → match/dedup against existing records → write to MATRIX sheet. The Cloud Run version replaces MATRIX writes with Firestore writes, and RAPID_CORE helpers with packages/core normalizers + matchers.
