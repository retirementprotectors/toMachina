Key Findings:
  • 81% are empty shells — default template with *Accounts, *Opportunities, Ai3 TEMPLATE. No real client data.
  • 33+ duplicate folder pairs — exact name dupes, cross-format dupes, misspelling dupes.
  • Joe + Laura Dickson Ai3 misfile — their template got copied into 6+ unrelated client folders.
  • 5 "ACF - undefined" folders — automation errors from watcher.
  • 16 loose root-level files — orphaned PDFs not in any client folder.
  • Source Documents is a dumping ground — 28 of 59 document types (47%) route there.

Document Taxonomy

  • File naming convention: PolicyNum- LASTNAME Carrier DocType YYYY-MM-DD

ProDash UI Document Linking

ACF subfolders must serve the ProDash UI directly:

Client Detail Page

  • DL photo (always show latest from Client subfolder)
  • Medicare card (for Medicare clients)
  • Voided check (for payment setup)
  • Ai3 (most recent quarterly)

Account Detail Page (by product type)

  • Most recent statement (from Account subfolder)
  • Policy document / application (from NewBiz subfolder)
  • Product-type-specific docs (MAPD enrollment, FIA illustration, etc.)

Case Detail Page

  • All related case documents (from Cases subfolder)

Config needed: Rules for which docs link to which UI locations, by product type. Proactive pull (scheduled scan) + Reactive pull (on new document arrival).

  • Identify patterns in what people actually filed
  • Validate proposed subfolder structure against real data

Data Phase 3 — Restructure

  • Update ACF Config with new 5-folder structure (NewBiz, Reactive, Cases, Account, Client)
  • Re-map _DOCUMENT_TAXONOMY acf_subfolder column to new structure
  • Create new subfolders in existing ACFs
  • Categorize and move existing documents into correct subfolders
  • Rename folders to standard ACF - First Last format

Cleanup Phase 4 — De-dupe + Consolidate

  • Merge 33+ duplicate folder pairs (pick winner with more data)
  • De-dupe documents within folders
  • Fix Joe + Laura Dickson misfiles
  • Clean up undefined/test/typo folders

App Phase 5 — ProDash UI Document Linking

  • Build document linking config (which docs → which UI locations)
  • Product-type-aware rules (Medicare vs Retirement vs Life)
  • Proactive pull (scheduled scan of ACF for linkable docs)
  • Reactive pull (on new document arrival via Watcher)
  • Client detail page: DL, Medicare card, voided check, Ai3
  • Account detail page: latest statement, policy doc

Infra Phase 6 — Update Routing Rules

  • Seed ACF Config routing_rules from _DOCUMENT_TAXONOMY
