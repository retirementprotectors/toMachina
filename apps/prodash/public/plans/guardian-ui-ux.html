Deliverable 1: Structural Health Script

New services/api/src/scripts/guardian-structural.ts

The script that answers: "What does our data ACTUALLY look like right now?"

Full Client Scan — ALL 5,197 (no sampling)

  • Field-by-field population rates: first_name, last_name, email, phone, dob, address, city, state, zip, client_status, household_id, agent_name, source, ghl_contact_id
  • Completeness score per client (% of recommended fields populated)
  • Distribution: how many clients are 100% complete? 75%? 50%? Below 25%?

Duplicate Detection (TRK-13548)

  • Group ALL clients by normalized(first_name) + normalized(last_name) + dob
  • Report duplicate clusters: count, doc IDs, which fields differ
  • Also check: same email across multiple clients, same phone across multiple clients

Carrier Normalization Audit

  • Read all 270 carriers + ALL accounts across every client subcollection
  • Map every account carrier_name against carriers collection
  • Report: exact matches, fuzzy matches (levenshtein < 3), complete mismatches
  • Produce carrier alias mapping suggestion

Collection Size Report

Every protected collection: doc count, last-modified sample. Full inventory of the database.

Output: Console report + Firestore (guardian_structural_reports/{id}) + JSON file for Damage Report HTML
