7-Step Submission Workflow
🚨 THE GOLDEN RULE

GS MUST APPROVE ALL PAPERWORK BEFORE: submitting to GI, Schwab, or RBC — transferring any funds — having clients sign (unless GS grants an exception). Violation = automatic NIGO + potential compliance issue.

🤖 Automated System Checks — ProDashX Enforces These

The NBX Investments pipeline in ProDashX has built-in system checks that automatically block case progression if these fail. They run at the Suitability and QC Review stages:

  • FIELD_PRESENT — blocks if net worth, annual income, risk tolerance, or investment objective is blank on CAF
  • LNW_LIMIT — automated check: liquid net worth within buffered product limits (15%/45% rules)
  • BI_UNIQUE — detects copy-pasted Best Interest rationale. If it matches another case, it fails automatically.
  • CORRECT_CUSTODIAN — flags if GWM forms reference RBC, or RBC forms reference Schwab/GI
  • AMOUNTS_MATCH — catches mismatched dollar amounts across forms (e.g., $226K on TAF vs $224K on UACT)

These are the same errors that caused the Stirratt and Clark NIGOs — now they're automated gates. You still need to catch them before upload, but the pipeline will flag them too.

NIGO Prevention — Real Errors from Real Cases
These Are Not Hypothetical — These Errors Already Happened at RPI

Every item below caused an actual NIGO or compliance flag. Mark Grams and Morgan Smick have flagged RPI cases for all of these. Learn them.

1. GI Paperwork Used on an RBC Case

From case: STIRRATT (original NIGO — Case #571086, flagged by Morgan Smick)

Submitted GI/Schwab advisory forms for a client going into an RBC brokerage account. Wrong forms for the wrong path.

For RBC brokerage cases, use only RBC forms: RBC CAF, RBC TAF, RBC IRA Adoption Agreement, RBC UACT, RBC Client Fee Listing. GI forms (IPC, IAA, GI Disclosure) are NOT used for RBC cases.

2. Wrong Dollar Amount on TAF Source of Funds

From case: STIRRATT (current packet error)

TAF shows $226,424.38 as the source amount. That is the client's CHECKING balance (from CAF Section 4). The actual IRA being transferred is $223,955 at Schwab account #31846959. These are completely different accounts.

The Source of Funds amount = the SPECIFIC ACCOUNT BALANCE being transferred. Find the correct account statement. Use that number. Not the checking balance, not the total retirement assets, not net worth.

3. Wrong Vendor on TAF Source of Funds

From case: STIRRATT (current packet error)

TAF says "Brokerage RBC" as the vendor/source. RBC is the RECEIVING firm, not the delivering firm. The money is coming FROM Charles Schwab.

Vendor = DELIVERING firm. The firm where the money currently sits. For Schwab transfers: "Charles Schwab." For Pershing transfers: "Pershing LLC." For direct rollovers from employer plans: plan name.

4. Blank Beneficiary Designation on IRA Adoption Agreement

From case: STIRRATT (current packet gap)

IRA Adoption Agreement Section 3 is completely blank. An IRA without a named beneficiary defaults to the estate — creating potential probate issues and tax problems for the heirs.

ALWAYS get beneficiary information before preparing the IRA Adoption Agreement. At minimum: Primary beneficiary name, relationship, SSN, DOB, percentage. If client doesn't know — don't leave it blank. Flag it and get an answer before sending to GS.

5. Primary Residence Value Missing from CAF

From case: STIRRATT (current packet gap)

CAF shows Primary Residence Value as blank, but Net Worth shows $1,412,898. The math only works if the residence value is ~$706K — but it's not written anywhere on the form. GS will flag this as an incomplete financial picture.

Get the residence value from the client during fact-finding. Verify the net worth math: Total Investable Assets + Residence Value − Liabilities = Net Worth. All three must be filled in and must add up correctly.

6. Date Mismatch Across Forms

From case: STIRRATT (current packet error)

Rep Acknowledgements page says CAF dated 03/17/2026. Client Acknowledgements page says CAF dated 02/17/2026. The client is acknowledging the wrong document date. GS will catch this.

After completing a packet, read every date on every page. All references to the CAF date must match. All references to the COMRA date must match. Use find-and-replace in your mind: search for every date box and verify consistency.

7. Document Alteration (Whiteout)

From case: CLARK — actual compliance flag, LPOA revoked (Case #570593)

Whiteout was detected on a signed document. This triggered a compliance hold. In a FINRA-regulated environment, even well-intentioned corrections to signed documents create audit flags and can be interpreted as document fraud.

Zero tolerance. No whiteout. No correction tape. No overwriting. If you make an error before signing: reprint and start over. If client has already signed and a correction is needed: new form, new client signature.

8. Funds Transferred Before Approval

From cases: CLARK + BUDD — both flagged by Mark Grams

$14,470 was transferred for Clark without GS approval. $100K sat uninvested for 27+ days for Budd because paperwork was incomplete. Transferring funds without completed, approved paperwork is a direct compliance violation.

No money moves until: (1) GS has approved the paperwork, (2) client has signed, (3) final submission is made. The order is fixed. No exceptions without written approval from Mark Grams.

9. Contradictory Investment Objectives on CAF

From case: STIRRATT (current packet issue)

Both "Preservation of Capital" AND "Growth and Income" are checked on the CAF. These are contradictory for the same client. A suitability reviewer will ask why a 68-year-old retiree with COMRA 35 has growth objectives.

Check ONLY the investment objective that matches the COMRA category and the Best Interest narrative. COMRA 35 = Conservative Growth. Appropriate objective: Preservation of Capital or Balanced Growth. Pick one. Remove the contradictory one.

10. Fee Inconsistency Across Forms

From cases: CLARK + STIRRATT

IAA/IPC shows one fee, TAF Best Interest narrative shows a different fee. For Clark, the TAF said 1.53% (incorrectly including ETF expenses) when the correct advisory fee is 1.40% (GWM 1.00% + GI 0.40%).

Standard fee formula: GWM Advisory (1.00%) + GI TPMM (0.40%) = 1.40% stated advisory fee. Internal ETF expenses (0.13–0.25%) are below-the-line costs, not advisory fees — do not add them to the stated fee. This number must be identical on: TAF Best Interest section, TAF page 3, IAA Schedule D, IPC Schedule D.

11. Generic / Copy-Pasted Best Interest Rationale
