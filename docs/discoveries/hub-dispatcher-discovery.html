Hub Dispatcher

CXO-aware intake router. One triage point, five destinations. Every incoming item lands in the right queue without JDM sorting manually.

The Problem

Incoming work arrives through multiple channels (Slack, email, MDJ chat, Dojo Reporter). Today, the intent classifier only knows deploy vs chat and the triage engine only routes to RAIDEN. Neither knows about MEGAZORD, VOLTRON, or MUSASHI. Data import requests sit in RAIDEN's fix queue. Creative briefs land as bug reports. JDM manually re-routes.

The Solution

Incoming item | v HUB DISPATCHER (keyword scoring + Haiku fallback) | +---> Data problem? ---> MEGAZORD (#megazord-intake / megazord@) +---> Client case? ---> VOLTRON (#voltron-cases / voltron@) +---> Creative request? ---> MUSASHI (Command Center) +---> Bug / broken thing? ---> RAIDEN (#the-dojo / dojo@) +---> Feature / build? ---> RONIN (Dojo intake) +---> Junk / spam? ---> Archive (auto-discard)
