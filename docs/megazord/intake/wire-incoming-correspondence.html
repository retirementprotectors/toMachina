Layer 1: Atomic Tools

Extract from watcher.js:1373-1493. Claude Vision analyzes all pages, detects document boundaries. Keep the proven prompt verbatim.

TRK-294: split-pdf

Extract from watcher.js:1503-1565. PyMuPDF via Python subprocess. Cloud Run needs Python + PyMuPDF in Docker image.

TRK-295: label-document

Taxonomy lookup + file labeling. Phase 1: reads _DOCUMENT_TAXONOMY from Sheets (bridge). Phase 2: Firestore.

TRK-296: SUPER_EXTRACT vision mode

Add mode: 'vision' to existing extract.ts. Extract extractMailData() and extractDataFromImages() from watcher.js. Keep proven Claude Vision prompts verbatim. Existing CSV mode unchanged.

Layer 2–3: Engine

TRK-292: SUPER_CLASSIFY

Composes: classify-boundaries → split-pdf → label-document. Single-doc passthrough if only 1 document detected.

TRK-291: Wire Executor — THE missing piece

executeWire(wireId, input, context) — loads wire definition, runs SuperTools sequentially, passes output between stages, tracks status per stage, writes wire_execution audit doc, pauses at approval stage, resumes on approval execution.

Layer 4: Infrastructure

TRK-297: Cloud Trigger

Eventarc on Firestore intake_queue onCreate → Cloud Run → executeWire(). Docker: Node.js + Python + PyMuPDF. Scales to zero. No timeout. Replaces launchd.

TRK-298: Approval Rewiring

Wire executor creates approval batch via toMachina POST /api/approval/batches (not archived GAS). Pauses wire at AWAITING_APPROVAL. Resumes on execute → SUPER_WRITE → ACF → notify.

Key Rules

  • Proven prompts are sacred. Extract Claude Vision prompts from watcher.js verbatim. Do NOT rewrite.
  • Old watcher is NEVER deleted. Disable launchd, don't uninstall. One command to roll back.
  • Taxonomy stays in Sheets (Phase 1). Don't block on Firestore migration for config.
  • Wire executor pause/resume: Firestore listener on approval_batch status, not polling.
