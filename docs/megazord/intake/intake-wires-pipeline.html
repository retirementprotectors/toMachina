Execution Architecture

intake_queue (Firestore onCreate)
  → wire-trigger (Cloud Function) — lightweight HTTP dispatcher
    → POST /api/intake/execute-wire (Cloud Run tm-api)
      → SUPER_PREPARE    — Download from Drive, PDF→PNG via poppler
      → SUPER_CLASSIFY   — Boundary detect, split, label
      → SUPER_EXTRACT    — Claude Vision reads pages, extracts data
      → SUPER_VALIDATE   — Qualification gate
      → SUPER_NORMALIZE  — 16 field normalizers
      → SUPER_MATCH      — 3-tier dedup (exact → fuzzy → new)
      → ⏸️ APPROVAL       — Human review in ProDash Notifications
      → SUPER_WRITE      — Approved data → Firestore
      → ACF_FINALIZE     — Route file to ACF + update document_index

Data Flow Contract

WireInput { file_id, file_ids, mode, _meta: { file_name, client_id, source } }
  ↓
PrepareOutput { image_paths[], file_name, file_path, taxonomy_types[], source_file_id, client_id? }
  ↓
ClassifyOutput { document_count, documents: [{ type, label, acf_subfolder, pages[] }] }
  ↓
VisionExtractOutput { client: {}, accounts: {}, mail_metadata: {}, confidence: {} }
  ↓
ValidateOutput { qualified: Record[], disqualified: { record, reason }[] }
  ↓
NormalizeOutput { records: Record[], total_changes }
  ↓
MatchOutput { matched: [{ record, tag, existing_id?, confidence }], summary }
  ↓
WriteOutput { batch: WriteRecord[], summary: { creates, updates, skips } }
  ↓
FinalizeOutput { routed, target_subfolder, document_index_updated, source_file_cleaned }
