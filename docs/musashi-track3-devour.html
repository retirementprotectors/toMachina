Why DEVOUR Is Track 3

"CONSUME gave MUSASHI a voice. OPERATE gave MUSASHI work. DEVOUR gives MUSASHI a mind."

Track 1 built the CMO Registry from scratch — 50+ tools registered, wire stubs defined, the domain claimed. Track 2 wired 3 Artisans live: Digital, Web, and Print. A content pipeline runs. An intake channel exists. MUSASHI produces on command.

DEVOUR is full creative intelligence. MUSASHI audits everything it owns. It maps what exists in print against what's missing in digital. It activates the remaining 2 Artisans (Social, Video). DEVOUR's Definition of Done is different from every other track. DEVOUR is never "complete." It is a state MUSASHI enters and never leaves. MUS-D12Brand Compliance Engine

Automated compliance checker for every Artisan output. Validates asset metadata against the brand guide before anything goes to Drive, WordPress, or C3. Violations sent to JDM DM — never silently corrected.

File Paths

packages/core/src/cmo/brand/compliance-engine.ts — NEW

Acceptance Criteria

  • checkBrandCompliance(asset: CmoAssetDescriptor): Promise<CmoBrandComplianceReport> exported
  • CmoAssetDescriptor: assetId, type, channel, colors?: string[], fonts?: string[], hasGeneratedLogo: boolean, copy?: string, artisan: string
  • CmoBrandComplianceReport: assetId, passed: boolean, violations: CmoBrandViolation[], checkedAt: Date
  • CmoBrandViolation: rule, description, severity: 'blocking' | 'warning'
  • Blocking checks: unapproved colors, generated logo, lorem ipsum in copy

  • Logs to console: asset type + pass/fail only (no PHI, no asset content)

Slack Notification Format

Brand compliance violation — {artisan} / {assetId} Violations: - [BLOCKING] Unapproved color: #ff0000 - [WARNING] Font not in approved stack Asset type: {type} | Channel: {channel} Checked at: {ISO timestamp} Action required: Review before publish.

  • Asset with hasGeneratedLogo: true → passed === false, logo blocking violation present
  • Asset with all approved colors, no logo → passed === true

Oct 1 – Dec 31: all non-AEP campaign entries are blocked with reason 'AEP_BLACKOUT'. No override. No exceptions. If MUSASHI is asked to schedule a non-AEP campaign in that window, the entry is created as 'blocked'. Artisan does not produce the asset until the blackout lifts.
