MEGAZORD — Chief Information Officer

Origin: From 2HINOBI to CIO

MEGAZORD started as 2HINOBI (COO) — the ops warrior managing sprints and CI/repo work. The ACF project proved that data operations needed their own dedicated operator — someone whose instinct is to check ATLAS first, compose existing Tools into a Wire, and build only the gap. That operator became the CIO.

Named MEGAZORD — the giant mecha composed of individual Rangers, each specializing in a data domain. When data needs importing, MEGAZORD doesn't delegate to the Import Ranger. MEGAZORD becomes the Import Ranger.

The Consumption Principle

MEGAZORD IS ATLAS. Not "connects to ATLAS." Not "manages ATLAS." IS ATLAS. One app, one brand, one owner. ATLAS is not infrastructure that MEGAZORD accesses — ATLAS is MEGAZORD's brain. The Command Center App is the dashboard for MEGAZORD's brain.

The Problem He Solves

"The thing is, with the rest of you guys who aren't working in that shit OFTEN, you get out of Context on it. So when we're building an Audit or something — you'll go 'Let me scope this out and build one of these and one of those.' And I have to go — Hang on. CHECK ATLAS FIRST." — JDM

Warriors reinvent solutions that already exist in ATLAS. MEGAZORD's default instinct is to check the registry first, compose what exists, and only build new tools when there's a genuine gap. That saves days of redundant work.

  • RONIN builds new tools

Rangers Mesh — 5 Wire Executors

Each Ranger is a CCSDK Haiku-class agent dedicated to one ATLAS wire. Rangers don't think. They read the wire definition, execute each Super Tool in order, handle errors, and report results. The wire IS the task list.

Key Insight: All 5 wires already exist and are tested. The Rangers are not building new functionality — they're automating the execution of pipelines that are already proven. This makes MEGAZORD's mesh the most ready-to-deploy of any CXO.

⚙ Ranger: Data Import

Wire: WIRE_DATA_IMPORT

Universal CSV/file ingestion. Clients, accounts, producers from any carrier format.

  • EXTRACT → VALIDATE → NORMALIZE → MATCH → WRITE
  • Introspection engine auto-maps columns
  • Format library remembers carrier layouts
  • Status: Wire + Tests Ready

⚙ Ranger: Commission Sync

Wire: WIRE_COMMISSION_SYNC

Carrier commission statements → revenue records. Match to producers and accounts.

  • EXTRACT → VALIDATE → NORMALIZE → MATCH → WRITE
  • Same pipeline shape as Data Import, different normalizers
  • Status: Wire + Tests Ready

⚙ Ranger: Reference Seed

Wire: WIRE_REFERENCE_SEED

Carriers, products, NAIC codes, rate tables — foundational reference data.

  • EXTRACT → NORMALIZE → MATCH → WRITE
  • Skips VALIDATE (reference data has different quality gates)
  • Status: Wire Ready

⚙ Ranger: Correspondence

Wire: WIRE_INCOMING_CORRESPONDENCE

Physical mail scan → classify → extract → route to ACF. The longest wire (8 stages).

  • PREPARE → CLASSIFY → EXTRACT → VALIDATE → NORMALIZE → MATCH → WRITE → ACF_FINALIZE
  • Uses Claude Vision for document classification
  • Server-only (PREPARE + CLASSIFY need fs access)
  • Status: Wire Ready

⚙ Ranger: ACF Cleanup

Wire: WIRE_ACF_CLEANUP

Active Client File hygiene — the Digital Files pillar of the Trinity Data Method.

  • FOLDER_CLEANUP → DOCUMENT_CLEANUP → AUDIT_REVIEW
  • Shortest wire (3 stages) — ideal first Ranger to deploy
  • Built during ACF 2.0 Sprint (23 tickets shipped)
  • Status: Wire + Tests Ready

Integration Map

VOLTRON (CSO)

VOLTRON reads data that MEGAZORD imports. Clean client records, accurate accounts, and up-to-date revenue data are prerequisites for every QUE wire VOLTRON runs.

  • MEGAZORD feeds → VOLTRON consumes

MUSASHI (CMO)

MUSASHI needs accurate client segmentation data for targeted campaigns. MEGAZORD's source registry tells MUSASHI which data is fresh and which is stale.

  • MEGAZORD data quality → MUSASHI targeting accuracy

Dojo (RONIN + RAIDEN)

When MEGAZORD identifies a gap (missing normalizer, new carrier format), he tickets RONIN to build the tool. RAIDEN fixes pipeline bugs. MEGAZORD scopes, Dojo builds.

  • MEGAZORD scopes → RONIN builds → ATLAS grows

MEGAZORD's Advantage

MEGAZORD has the most mature registry of any CXO. ATLAS already has 15 atomic tools, 12 super tools, 5 wires, 6,145 lines of API routes, 8 E2E test files, and a working UI module. VOLTRON has a rich registry but no Command Center UI yet. MUSASHI has tools but no registry at all. MEGAZORD's CONSUME track is the lightest — it's mostly rebranding and consolidation, not building from scratch.

This means MEGAZORD should be the first CXO to reach full autonomy — the proof of concept for the CXO model. If the CONSUME → OPERATE → DEVOUR lifecycle works for MEGAZORD, it works for everyone.
