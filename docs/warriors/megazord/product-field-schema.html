Data Migration Protocol — 13 Steps

Every product type goes through this. No shortcuts. Three JDM gates.

1

Lock Product Schema

Define fields per type/subtype. JDM approves via interactive tool.

2

Create Dropdowns / Lookup Tables

Populate products collection. Carrier, product_type, subtype become constrained lookups.

3

Update TypeScript Account Interface

Add new fields, remove dead ones. Type system becomes the enforcer.

4

Update Frontend Rendering

Show/hide fields by product type. Dropdowns not free text. No invalid field combos.

5

Code-First Deploy

Push code that reads BOTH old and new field names. Deploy. Verify live.

6

Dedup

Match records against the schema. Merge duplicates. Schema IS the dedup key.

7

Dry Run — JDM GATE

Sample before/after on migrated + deduped data. JDM reviews.

8

Execute Migration

Merge fields, delete dead fields, atomic writes. Batch 450.

9

Verify

Spot check 5 records + global scan. Zero dead fields remaining.

10

Cleanup

Remove old field fallbacks from code. Deploy.

11

Communicate — JDM GATE

Draft team message. JDM approves. Send to #the-dojo.

12

Monitor

3-5 business day watch window. Team reports issues.

13

Close

Migration officially complete. Schema locked.
