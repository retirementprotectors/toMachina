Builder 1: ATLAS Tools

TRK-266: Firestore acf_config

  • Types in packages/core/src/acf/types.ts: ACFConfig, ACFCreateInput/Output, ACFRouteInput/Output, ACFAuditOutput, ACFRebuildInput/Output, ACFStatus
  • Seed with current hardcoded values from watcher.js:2557-2574
  • Security rules: admin-only write, all RPI users read

TRK-267: acf-create

  • Extract createACFFolder() + shareFolderWithTeam() from watcher.js:2664-2725
  • Reads config from Firestore acf_config (not hardcoded)
  • Creates Drive folder + subfolders + Ai3 copy + shares with domain
  • Writes acf_folder_id + acf_folder_url to Firestore client doc

TRK-268: acf-route & TRK-269: acf-status

  • acf-route: routes documents to specific ACF subfolder. Generalizes copySourceDocsToACF(). If ACF missing, flags acf_missing: true.
  • acf-status: reads acf_folder_id from Firestore, verifies Drive access, counts subfolders/docs, checks Ai3. Replaces Sheets-based getClientACFLink().

TRK-270 + TRK-271: Audit + Rebuild

  • acf-audit: scans ALL clients. Categorizes: with_acf, missing, broken_links, incomplete, orphaned. Returns client ID arrays.
  • acf-rebuild: 3 modes — create_missing, fix_broken, full_rebuild. Supports dry_run.

TRK-277: Household-Aware Logic

  • acf-create checks household_id on client doc before creating folder
  • If household member → creates under household ACF with per-member subfolder
  • Respects shipped TRK-13308 + TRK-13309 patterns

Builder 2: UI

TRK-273: Contact Detail — ACF Section

  • Status badge: Complete / Partial / Missing
  • Subfolder accordion with document counts
  • Expand subfolder → document list (name, date, type)
  • Buttons: Open in Drive, Upload to ACF (file picker + subfolder select), Create ACF (if missing)

TRK-274: Contact Grid — ACF Status Icon

  • Green check (complete) / Yellow warning (partial) / Red X (missing)
  • Tooltip: subfolder count, document count, last updated

TRK-275 + TRK-276: Admin Pages

  • Config tab: edit template, subfolders (add/remove/reorder), naming pattern, toggles, Test Config button
  • Audit page: Run Audit → results table (Missing/Broken/Incomplete/Orphaned tabs) → select clients → choose mode → dry run → execute

Key Notes

  • Zero MATRIX dependency. All tools read/write Firestore only.
  • Drive API auth: service account in Cloud Run, ADC in dev.
  • Orphaned detection is expensive (scans Drive). Opt-in flag, not default.
  • Never auto-delete Drive folders. Flag for manual review only.
