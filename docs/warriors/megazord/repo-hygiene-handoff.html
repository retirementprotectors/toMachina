10 Atomic Tools — Repo Audit + Hygiene

Each tool is a single API endpoint in services/api/src/routes/system-synergy.ts. Each is registered in ATLAS. Each returns structured JSON.

TOOL-01

session-inventory

What: List all Claude Code sessions with metadata — warrior name, working directory, date, file size, subagent count, active/stale classification.

Reads: ~/.claude/projects/{proj}/{id}.jsonl + ~/.claude/sessions/*.json

Returns: { sessions: [{ id, warrior, cwd, date, size_mb, subagent_count, age_days, status }], total, stale_count }

Why: Manual session e35411cd found stale worktrees and orphaned temp dirs. This detects them automatically.

TOOL-02

memory-inventory

What: List all MEMORY.md entries across all projects with staleness scoring. Flag entries older than 21 days unconfirmed.

Reads: ~/.claude/projects/*/memory/*.md + MEMORY.md index files

Returns: { entries: [{ file, type, name, age_days, stale }], total, stale_count, orphaned_count }

Why: Session 9f018025 deleted stale memory snapshots manually. This catches them before they confuse warriors.

TOOL-03

claude-md-diff

What: Show what changed in any CLAUDE.md over the last N days using git history.

Reads: git log -p on CLAUDE.md + ~/.claude/CLAUDE.md

Returns: { changes: [{ date, commit, lines_added, lines_removed, summary }] }

Why: Deferred maintenance sprint (DM-01, DM-06) required manually auditing CLAUDE.md staleness. This automates the check.

TOOL-04

brain-health

What: Per-warrior brain.txt stats — lines, last updated, PHI scan, size, extraction status.

Reads: /home/jdm/Projects/dojo-warriors/*/brain.txt + soul.md + spirit.md

Returns: { warriors: [{ name, brain_lines, brain_size, brain_last_updated, soul_lines, spirit_lines, phi_detected }] }

Why: Session 9f018025 found VOLTRON and RONIN have zero brain.txt content. Extraction scripts exist but were never run. This surfaces gaps.

TOOL-05

knowledge-query

What: Query Firestore knowledge_entries by type, confidence, warrior, date range.

Reads: Firestore knowledge_entries collection

Returns: { entries: [{ id, type, confidence, warrior, source, created, content_preview }], count }

Why: The Learning Loop pipes exist but no one can see what's in them. This is the query interface.

TOOL-06

session-env-audit

What: Map session-env directories to sessions, find orphans (env dirs with no matching session), find stale sprint directories.

Reads: ~/.claude/session-env/ + ~/.claude/projects/ + .claude/sprints/

Returns: { orphaned_envs: [], stale_sprints: [], temp_project_dirs: [], total_session_envs, active_sprints }

Why: Session 9f018025 found 19 stale sprint dirs and temp RONIN project dirs. These accumulate every week.

TOOL-07

duplicate-detector

What: Find duplicate/conflicting entries across MEMORY.md files, duplicate HTML docs, and redundant config files.

Reads: ~/.claude/projects/*/memory/*.md + docs/**/*.html + config files

Returns: { duplicates: [{ file_a, file_b, similarity, type }], count }

Why: Session 9f018025 found junk copies of docs, duplicate knowledge-promote scripts, and stale discovery docs that confused warriors.

TOOL-08

warrior-roster

What: Which warriors have tmux sessions on MDJ_SERVER, queue depth per warrior, last activity timestamp.

Reads: tmux ls on MDJ_SERVER + /home/jdm/dojo/queue/*.json + Firestore dojo_warriors

Returns: { warriors: [{ name, tmux_active, queue_depth, last_activity, uptime }] }

Why: JDM needs one-call visibility into who's running and who's idle.

TOOL-09

deploy-status

What: Current state of all CI runs, last successful deploy timestamp, pending PRs, broken tests.

Reads: GitHub Actions API + Cloud Run revision list + Firestore deploy logs

Returns: { last_deploy: { sha, time, status }, pending_prs: [], ci_runs: [], broken_tests: [] }

Why: Session 9f018025 found broken E2E tests and stale PRs. This surfaces them proactively.

TOOL-10

hook-audit

What: Verify hookify symlinks across all projects, report missing/broken rules, check service health on MDJ_SERVER.

Reads: Hookify symlinks in all active projects + systemctl status on MDJ_SERVER services

Returns: { projects: [{ name, rules_linked, rules_missing, rules_broken }], services: [{ name, status, uptime }] }

Why: Session 9f018025 found memory-guardian.service FAILING (wrong path) and dojo-watchdog disabled. This catches drift.

4 Super Tools — Composites

Each Super Tool composes multiple atomic tools into a single meaningful operation. Registered in ATLAS as Super Tools.

SUPER-01

platform-health

Composes: session-inventory + warrior-roster + deploy-status + hook-audit

Returns: Full platform status — warriors, deploys, hooks, sessions — in one call.

When to call: JDM says "check the immune system." VOLTRON gets asked "how's the platform?" Any warrior starts a session and wants current state.

SUPER-02

knowledge-pipeline-status

Composes: knowledge-query + memory-inventory + brain-health + claude-md-diff

Returns: Is the learning loop flowing? Where's it stuck? Which warriors have stale brains?

When to call: Debugging why a warrior "forgot" something. Weekly health check. After new warrior onboarding.

SUPER-03

session-forensics

Composes: session-inventory + session-env-audit + duplicate-detector

Returns: Orphaned sessions, stale sprint dirs, duplicate configs, temp worktree remnants.

When to call: Disk space issues. Warrior confusion from stale state. Weekly cleanup.

SUPER-04

warrior-readiness

Composes: warrior-roster + brain-health + memory-inventory

Returns: Is each warrior's context fresh and complete? Who has gaps? Who's stale?

When to call: Before launching a RONIN sprint. Before starting a MEGAZORD data operation. Session start briefing.

4 Wires — Automated Execution

Wires are scheduled or triggered compositions. They run without human intervention. Each is a systemd timer or Claude Code hook on MDJ_SERVER.

WIRE-01

WIRE_KNOWLEDGE_PROMOTE

Schedule: Daily 4am (systemd timer)

Flow: entity-extractor → knowledge-query (confidence ≥ 0.8) → claude-md-diff → auto-write to CLAUDE.md + Slack digest to JDM DM

What changes vs today: Currently knowledge-promote only sends a Slack DM. This wire adds the actual CLAUDE.md auto-write that was never ported from macOS v2.

WIRE-02

WIRE_BRAIN_SYNC

Schedule: Daily 2am (systemd timer)

Flow: session-inventory (find new sessions since last run) → brain-export (extract transcripts, PHI-redact) → brain-health (verify append succeeded)

What changes vs today: Brain.txt is only appended manually or by extraction scripts that are never run. This automates it. VOLTRON and RONIN finally get brain files.

WIRE-03

WIRE_PLATFORM_AUDIT

Schedule: Weekly Sunday 3am (systemd timer)

Flow: platform-health + session-forensics + duplicate-detector → auto-delete orphaned session-envs older than 30 days → auto-archive stale sprint dirs → cleanup report to JDM DM

What changes vs today: SHINOB1 did this manually on April 5-6. This wire does it every Sunday automatically. No more 48-inactive-repo pileups.

WIRE-04

WIRE_WARRIOR_BRIEFING

Trigger: SessionStart hook (not scheduled — fires when any warrior session starts)

Flow: warrior-readiness (check this warrior's context freshness) + cross-warrior knowledge-query (what did OTHER warriors learn since last session?) → inject briefing into session

What changes vs today: Warriors start cold. No cross-pollination. MEGAZORD doesn't know what SHINOB1 did last night. This fixes that.
