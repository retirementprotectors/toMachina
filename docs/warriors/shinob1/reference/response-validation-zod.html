Response Validation (Valibot)

Non-blocking runtime validation at every API call site. Safety net for type mismatches that TypeScript can’t catch across HTTP boundaries.

The Problem

TypeScript types evaporate at the HTTP boundary. The FORGE React crash (2026-03-22) proved it: API returned { pending: TrackerItem[] }, frontend expected { pending: number }, React crashed rendering an array as a text node.

170 fetchWithAuth calls across 52 UI modules — zero runtime validation on any of them.

Key Decisions

Library
Valibot v1
7KB tree-shaken vs Zod’s 57KB. Same DX, same TS inference.
Schema Location
packages/core/src/schemas/
Next to types they validate. Shared by frontend + backend.
Validation Mode
Non-blocking
console.warn in dev, silent in prod. Never crashes.
DTO Dependency
Removed (JDM-approved)
Existing core types are sufficient. DTOs can tighten later.
