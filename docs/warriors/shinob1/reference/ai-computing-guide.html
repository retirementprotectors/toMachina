GPU — The Math-Specialized Brain

A GPU (Graphics Processing Unit) has thousands of simple cores designed for massive parallel math — exactly what AI models need. It doesn't replace the CPU; it sits alongside it in a PCIe slot and handles AI workloads when the CPU sends them over.

CPUGPU
Cores16 (complex)Thousands (simple)
Good AtComplex decisions, logic, running appsSame calculation on millions of numbers simultaneously
Analogy16 PhDs solving hard problems4,000 calculators all multiplying at once
Needed ForEverything (OS, apps, networking)AI model inference (running LLMs locally)
The CPU always stays. The GPU can't run by itself. The CPU runs the server, OS, and all applications. The GPU is a specialist that only activates when the CPU hands it AI work. No CPU = server doesn't boot.

How They Work Together (The Spillover Effect)

An AI model needs to fit somewhere in memory to run. If it fits entirely in VRAM, it runs fast. If it's too big, it spills into system RAM — which works, but at a fraction of the speed.

Scenario A: Model Fits in VRAM (A100 40GB + Llama 70B at ~40GB) +---------------------------------------+ | GPU VRAM (40GB) | | [====== Entire model here ======] | <-- FAST (2,039 GB/s) | All processing on GPU | +---------------------------------------+ System RAM: barely used for the model Result: 25-40 words/sec Scenario B: Model Spills to System RAM (P40 24GB + Llama 70B at ~40GB) +---------------------------------------+ | GPU VRAM (24GB) | | [==== 60% of model ====] | <-- FAST (346 GB/s) +---------------------------------------+ | spillover | +---------------------------------------+ | System RAM (64GB) | | [== 40% of model ==] | <-- SLOWER (50 GB/s) +---------------------------------------+ GPU constantly pulling from system RAM Result: 5-15 words/sec
Bottom line: More system RAM doesn't make AI faster — it just gives the overflow somewhere to land instead of crashing. VRAM is the speed bottleneck. The real question is always: does the model fit in VRAM?

How It All Fits in the T440

Dell PowerEdge T440 — Motherboard Layout +--------------------------------------------------+ | | | [CPU Socket 1] [CPU Socket 2] | | (Installed) (Empty) | | | | [A1][A2][A3][A4] [A5][A6][A7][A8] <-- DIMM Slots (CPU 1) | 16 16 32 -- -- -- -- -- 3 of 8 used | | | [B1][B2][B3][B4] [B5][B6][B7][B8] <-- DIMM Slots (CPU 2) | Requires 2nd CPU to activate | | | | [PCIe Slot 1] x4 <-- Available (wired to CPU1) | | [PCIe Slot 2] x4 <-- Available (wired to CPU1) | | [PCIe Slot 3] x16 <-- GPU goes here (wired to CPU2) !! | | [PCIe Slot 4] x8 <-- RAID controller (occupied) | | [PCIe Slot 5] x8 <-- Available (wired to CPU2) | | | | [SATA Ports] <-- Hard drives / SSDs | | | +--------------------------------------------------------------+ !! CRITICAL: Slot 3 lanes are hardwired to CPU Socket 2. With Socket 2 empty, Slot 3 is electrically dead. The P40 physically fits but will NOT be detected by the OS. CPU2 must be installed BEFORE the GPU.

CPU, RAM, GPU, and storage all coexist on the same motherboard. They're independent components that work together. Adding a GPU doesn't remove anything — it fills an empty slot. Exception: the T440's x16 slot (Slot 3) is hardwired to CPU Socket 2 — the GPU cannot be used until a second CPU is installed.

Why NVIDIA Has a Monopoly

NVIDIA built CUDA — the software layer that every AI framework (PyTorch, TensorFlow, everything) is built on top of. AMD and Intel make GPUs too, but switching to them means rewriting all the software. Nobody wants to do that.

NVIDIA got the lock-in early and never let go. Every data center, every AI company, every cloud provider is buying NVIDIA GPUs. Their CEO Jensen Huang is essentially the gatekeeper of AI infrastructure worldwide.

The Mesh Layer — Where It Gets Powerful

The basic hybrid routes whole conversations to either local or Claude. The Mesh Layer goes further: local Llama handles the conversation for free, but when it needs to do something (send a Slack message, query Firestore, pull a document), it hands off to a Claude Code SDK agent that executes the action and returns the result.

The user never sees the handoff. They talk to one AI. Two brains work behind it.

User asks TBD.ai: "Send Vince the Mineart account summary on Slack" | Local Llama (free, instant) Understands the request, formats the intent | Can I do this alone? +--------+---------+ | | YES NO — needs to act on systems (just Q&A) | | Mesh Router dispatches | Claude Code SDK Agent Answer | (free) Agent executes: +---> MCP: Slack (send message) +---> MCP: Drive (pull account data) +---> Firestore (query client record) | Returns result to Llama | Llama formats response for the user (free)

Why This Matters

Without the Mesh, local Llama is a brain in a jar. It can think and talk but can't do anything. The Claude Code SDK gives it hands.

ApproachConversation CostAction CostCan Act on Systems?

The math: 90% of a conversation is back-and-forth clarification, context-building, and formatting. That's all free on Llama. You only pay Claude tokens for the 10% that requires tool execution. Biggest potential cost reduction of any architecture change.

The "Mid Route" Build (Current Plan)

Phase 4 (CPU2 + heatsink, ~$60) must come first — it activates PCIe Slot 3 so the GPU is detectable. Phase 5 (P40 GPU) is the AI unlock. If it proves the concept, Phase 6-7 (more RAM + A100) doubles down.

CPU2 is a prerequisite, not optional. The Tesla P40 physically installs in PCIe Slot 3 (the only x16 slot on the T440). That slot's lanes are hardwired through CPU Socket 2 on the motherboard. With Socket 2 empty, Slot 3 has zero active PCIe lanes — the GPU will not POST, will not appear in lspci, and CUDA will not detect it.

Parts needed before the GPU purchase:
  • Intel Xeon Silver 4110 (same model as Socket 1) — ~$40 on eBay
  • Dell T440 CPU2 heatsink, P/N 0FHPP8 — ~$20 on eBay
  • Move 1 existing RAM stick from CPU1 DIMMs to a CPU2 DIMM slot (required for CPU2 to POST)

Glossary

TermWhat It Means
CPUCentral Processing Unit. The general-purpose brain. Runs OS, apps, networking. MDJ1 has 16 cores.
GPUGraphics Processing Unit. Specialized math brain with thousands of simple cores. Runs AI models. Has its own built-in VRAM.
RAMSystem memory. DIMM sticks in the motherboard. Where the server holds data it's actively working with. 64GB on MDJ1.
VRAMVideo RAM. Built into the GPU card, not upgradeable. Where AI models sit during processing. Speed bottleneck for AI.
ECCError-Correcting Code. Self-healing memory that detects and fixes errors. Required for servers.
RDIMMRegistered DIMM. Server-grade RAM module. Required for T440 (desktop UDIMM won't fit).
DDR4The current generation of RAM technology. DDR5 is newer but T440 uses DDR4.
PCIe SlotPhysical slot on the motherboard where GPUs (and other expansion cards) plug in.
CUDANVIDIA's software layer that lets AI frameworks talk to GPU hardware. The reason NVIDIA has 90% market share.
LLMLarge Language Model. The type of AI that Claude, GPT, and Llama are. Trained on text, responds in text.
OllamaFree tool that manages local AI models. Install once, download models with one command, run them. Hides all complexity.
Open WebUIFree ChatGPT-like web interface. Connects to Ollama. Gives your team browser-based AI access.
InferenceRunning a trained model to get answers. As opposed to "training" (teaching the model). You only do inference.
Parameters (B)Model size. "70B" = 70 billion parameters. Bigger = smarter but needs more VRAM.
QuantizationCompressing a model to use less VRAM at slight quality loss. A 70B model can be squeezed to fit smaller GPUs.
TokensHow AI measures text. ~4 characters = 1 token. API pricing is per-token.
TailscaleVPN service that connects MDJ1 to all RPI devices securely, regardless of location.
NVIDIAThe company that makes ~90% of all AI GPUs. CEO: Jensen Huang.
