The Problem

"In an opportunity: The assignee and the agent should not be the same person. Assignees should be the administrative staff working the opportunity: Nikki Gray, Angelique Bonilla, Susan Kaelin."
Nikki Gray — #the-dojo, April 2

Assignee Selection Logic

The assignee dropdown should show admin/service team members. Two approaches:

Option A: Hardcoded list (fast, simple)

Nikki Gray, Angelique Bonilla, Susan Kaelin. Hardcode the 3 emails. Update when the team changes. Shipping in minutes.

Option B: Role-based query (scalable)

Query users where role is not OWNER/EXECUTIVE (admin staff). More maintainable. Automatically adapts as team grows.

Recommendation: Option B with a fallback

Query users with service/admin roles. If fewer than expected results, fall back to showing all non-owner users. The users collection already has role field — just needs the right filter. For the initial version, filter by: role NOT IN ['OWNER', 'EXECUTIVE'] OR a specific can_be_assignee: true flag.
