API

BUILDER 01 — Token + Outbound Webhook

TRK-13651TRK-13653

Files: services/api/src/routes/comms.ts, services/api/package.json

TRK-13651: Access Token Endpoint

POST /api/comms/tokenGenerate Twilio Access Token with VoiceGrant. Identity = authenticated user email. TTL 3600s. Returns { success, data: { token, identity } }

TRK-13653: Outbound Voice Webhook

POST /api/comms/webhook/voiceTwiML App callback. Reads To + From params, returns <Dial> TwiML with 888 caller ID + recording. Logs outbound call to Firestore.
<Response>
  <Dial callerId="+18886208587" record="record-from-answer-dual">
    <Number statusCallbackEvent="initiated ringing answered completed"
            statusCallback="/api/comms/webhook/call-status">{To}</Number>
  </Dial>
</Response>

Auth bypass required: Webhook routes (/api/comms/webhook/*) must skip Firebase auth middleware — Twilio calls them directly.

BUILDER 02 — Inbound Webhooks + Recording

TRK-13655TRK-13665TRK-13650

TRK-13655: Inbound Voice Webhook

POST /api/comms/webhook/voice-incoming888 incoming calls. Phase 1: route to josh@retireprotected.com. Returns <Dial><Client> TwiML with fallback voicemail. Logs to Firestore.
<Response>
  <Dial record="record-from-answer-dual">
    <Client>josh@retireprotected.com</Client>
  </Dial>
  <Say>We're sorry, no one is available. Please leave a message.</Say>
  <Record maxLength="120" action="/api/comms/webhook/voicemail" />
</Response>

TRK-13665: Inbound SMS Webhook

POST /api/comms/webhook/sms-incoming888 incoming texts. Extracts From, Body, MessageSid, media. Looks up client by phone. Logs to Firestore communications. Returns empty TwiML (no auto-reply).

TRK-13650: Recording + Call Status

POST /api/comms/webhook/recording-statusRecording complete callback. Updates Firestore doc with recording_url, duration, SID.
POST /api/comms/webhook/call-statusCall state changes (initiated → ringing → answered → completed). Updates status + duration in Firestore.
