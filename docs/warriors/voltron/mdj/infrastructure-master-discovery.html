Bot Loop Prevention

Prevent the dispatcher from sending messages that trigger itself in an infinite loop.

Audit trail posts by route() loop back as bot messages → absorbed by layer 1. No infinite loop possible.

CI Spam Dedup

Prevent duplicate CI failure notifications flooding the war room.
What works: pendingCI.delete(prNumber) after first failure report prevents the poll loop from double-reporting. trackPR() guards against double-tracking: if (pendingCI.has(prNumber)) return.
What's incomplete: reportedFailures: Set<number> declared at git-monitor.ts:32 but never populated or read — dead code. If trackPR is called with the same PR from two different Slack messages before the first poll fires, the second is silently dropped. Dedup works in practice but the intended mechanism (reportedFailures) is unwired.
src/dojo/git-monitor.ts
Partial

False LandedIt Prevention

Prevent incorrect #LandedIt! posts when the wrong deploy run is matched.
What works: pollDeployForMerge() polls every 15s for up to 10 min, checks gh run list --branch main --event push --limit 1, only posts on status=completed, conclusion=success. Merge scanner correctly identifies new commits and clears merged PRs from pendingCI.
Real race condition: checkDeployStatus() fetches the most recent push-triggered run on main — NOT the run for the specific commit. If merge A and merge B land within minutes, merge A's poller can pick up merge B's deploy run and post a false #LandedIt for the wrong commit. The commits array is passed in but only used for Slack message text, not to filter the GitHub Actions query. Fix needed: filter gh run list by head SHA, not just branch + event.
src/dojo/git-monitor.ts
