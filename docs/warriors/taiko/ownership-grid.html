The Drum’s Weekly Rhythm

TAIKO’s default operating cadence. Work proactively 1 → 5 through the week. Reactive fires interrupt but don’t replace the rhythm. The drum doesn’t stop between beats.

The rule: Every signal has to reach. That means checking the pipes before anything else — not after something breaks. The rhythm exists so the checks happen before the team notices a problem, not after they do.
1

Twilio Health Check — A2P, Deliverability, Error Rates

Carrier status for SMS and voice. A2P campaign registration current? Error rate on outbound SMS? Any new carrier flags? Twilio Console → Error Logs. First check every week: if SMS is blocked, nothing else matters.

2

SendGrid Deliverability Audit — Bounce Rates, DMARC / SPF / DKIM Health

Open rate, bounce rate, spam rate for the week. DMARC alignment passing? SPF/DKIM passing on all domains? Suppression list growing unexpectedly? Anything silently failing?

3

Internal Comms Check — Google Meet / Chat Usage + RPI Connect Bug Reports

Is the team using RPI Connect or routing around it? Any Google Chat integration errors? Meet recordings syncing to client records? Any team complaints in #taiko or #the-dojo?

4

Alerts Monitoring — Routing Accuracy, Noise vs. Signal

Are P0 alerts reaching JDM DM? Are non-P0 alerts staying out of JDM DM? Are warrior notifications routing to the right channels? Fatigue is how a P0 gets missed.

5

Sprint Work — ZRD-COMMS Tickets in Flight

Current sprint progress. Which tickets are in-flight? What’s blocked? What’s shipping this week? COMMS-001 (A2P unblock) is Job #1 until it closes.

Escalation rule: If any pipe is DEGRADED or BLOCKED, TAIKO posts in #the-dojo and opens a ZRD-COMMS ticket before moving to the next rhythm item. The drum doesn’t move on while a beat is missing.

TAIKO App — The Pipes Registry

TAIKO’s Future Warrior App

Every CXO in The Machine wields three weapons: a Registry, a Mesh, and a Dojo. This tab scopes TAIKO’s App — the registry of tools, super-tools, wires, health monitors, and the Beats mesh. Nothing built yet. This is the target architecture.

The architecture frame: MEGAZORD is ATLAS. VOLTRON is QUE. MUSASHI is the CMO Registry. TAIKO is the Pipes Registry. Every channel, every carrier, every webhook, every compliance gate is a registered tool. The App makes them visible, composable, and health-checked.
5a — Tools — Atomic Capabilities

The atomic building blocks TAIKO owns. Each does exactly one thing.

twilio-send-sms
Outbound SMS to a single recipient. A2P campaign gating enforced before send. Logs to comms_events Firestore collection. TCPA opt-in check is a hard gate — no opt-in = no send.
twilio-make-call
Outbound voice call. Accepts recipient number, agent caller ID, optional TwiML instructions. Returns call SID for tracking. Recording enabled by default.
twilio-webhook-inbound-sms
Process inbound SMS webhook from Twilio. Match to client record by phone number. Log to comms_events. Route to owning agent notification queue.
twilio-webhook-inbound-call
Process inbound voice webhook. IVR routing logic. Agent assignment. Return TwiML response. Trigger recording. Log call metadata to Firestore.
twilio-get-recording
Fetch recording audio URL and metadata for a completed call by call SID. Returns duration, status, media URL.
twilio-a2p-status-check
Query Twilio A2P 10DLC campaign registration status. Returns current state — APPROVED / PENDING / BLOCKED — failure reason if blocked, carrier audit data.
sendgrid-send-email
Send a single transactional email. Accepts recipient, subject, template ID or body, merge vars. Suppression list check is a hard gate. Returns message ID for delivery tracking.
sendgrid-send-bulk
Batch email send. Accepts recipient list with per-recipient merge vars. Chunked into SendGrid API limits. Returns per-recipient acceptance status. For campaign delivery — MUSASHI authors, TAIKO sends.
sendgrid-get-stats
Fetch SendGrid deliverability stats for a date range. Returns delivered, bounced, spam reported, opened, clicked. Core of the weekly deliverability audit.
sendgrid-check-suppression
Check if an email address is on any SendGrid suppression list. Returns suppression reason if found.
meet-create-meeting
Create a Google Meet event. Returns join URL, calendar event ID, meeting code. Wraps mcp__rpi-workspace__meet_create_meeting. Can pre-associate with a Firestore client record.
meet-get-transcript
Fetch transcript for a completed Google Meet session. Returns full text with speaker attribution. Wraps mcp__rpi-workspace__meet_get_transcript.
chat-post-message
Post a message to a Google Chat space. Accepts space ID, message text, optional thread key for replies. Wraps mcp__rpi-workspace__send_message.
alert-fire
Core routing dispatcher. Accepts payload — severity, event type, message, source. Applies routing rules table, dispatches to correct channel(s). Writes audit record to Firestore alert_log.
notification-write
Write a notification record to Firestore notifications collection with proper schema — target_user, type, message, urgency, read: false. The in-portal notification center reads from this collection.
5b — Super-Tools — Composed Capabilities

Atomic tools composed into intelligent, multi-step capabilities.

SUPER_COMMS_SEND
Route a message to the right channel based on user communication preference + message urgency. Input: client_id, message, urgency level. Checks client preference — SMS vs email vs chat — checks opt-in status, selects channel, dispatches. Single entry point for all outbound communication decisions.
SUPER_CLIENT_OUTREACH
Multi-channel client contact with fallback sequencing. Attempt 1: preferred channel. If undelivered after N minutes, Attempt 2: fallback channel — e.g. call → fallback SMS → fallback email. Logs each attempt to comms_events. Returns first successful delivery or exhausted-with-log failure.
SUPER_ALERT_ROUTE
Parse severity and event type, apply routing rules table, deliver to the right channel / warrior / person. Input: raw event payload. Outputs: delivery confirmation + alert_log write. P0 goes to JDM DM in under 60 seconds. Every event has a paper trail.
SUPER_A2P_HEALTH
Full Twilio A2P posture check. Queries campaign registration status, CTA verification status, opt-in compliance state, carrier rejection rate over the last 7 days. Returns structured health report: HEALTHY / DEGRADED / BLOCKED with specific failure reasons. TAIKO’s weekly SMS compliance instrument.
SUPER_DELIVERABILITY_AUDIT
Full SendGrid + DMARC deliverability audit. Fetches bounce rate, spam rate, delivery rate. Checks DMARC/SPF/DKIM alignment on retireprotected.com. Checks suppression list growth rate. Returns pass/fail per dimension with specific remediation for any failures.
