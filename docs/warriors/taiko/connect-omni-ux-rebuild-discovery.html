The Teardown Verdict

What's Wrong Today
  • 🚫
    CONNECT is a single 824-line god-file with no real 2-pane layout ConnectPanel.tsx renders all three tabs — channel list, people directory, and Meet — in one monolith. Selecting a channel causes a full-panel swap that nukes the left rail. There is no persistent 2-pane. It feels like a mobile app forced onto desktop. Real messaging UX keeps the list and the thread visible at the same time. ConnectPanel.tsx:480-488 — if (activeChannel !== null) return <ChannelChatView> replaces the entire panel
  • 🚫
    OMNI has no conversation view — every tab starts a fresh compose CommsModule/index.tsx routes each tab to a standalone CommsCompose. There is no concept of "view Mary Johnson's SMS thread." The Log tab shows history but has no way to jump into a conversation. Agents working a client have zero continuity across contacts. CommsModule/index.tsx:235-238 — all three content tabs render CommsCompose with no thread state
  • 🚫
    OMNI's Call tab is compose-first — no call history above the dialer The dialer appears immediately with no recent-calls context. Agents have no quick-redial, no missed-call indicator, no way to know who called before picking up the next one. The active call state machine is sound — window.__activeTwilioCall + callbacks — but that's an implementation detail, not a UX feature. CommsCompose.tsx:60-100 — dialer pad is the first and only element in the call channel
  • 🚫
    CONNECT People tab hides actions behind hover — zero mobile discoverability PersonCard uses opacity-0 group-hover:opacity-100 to hide Chat / Meet / Call buttons entirely at rest. On touch devices these actions are invisible and unreachable. New desktop users have no way to discover the actions exist without accidentally hovering. ConnectPanel.tsx:644-646 — opacity-0 group-hover:opacity-100 on the action row
  • 🚫
    CONNECT channel rows show phantom last-message data that never populates ChannelData carries unreadCount, lastSender, lastMessage, and timestamp fields — all initialized to empty strings and 0. The Google Chat spaces list endpoint returns space metadata only; getting a per-space last-message preview requires a separate messages call per space that was never wired. The result looks like a permanent loading skeleton. ConnectPanel.tsx:456-476 — ChannelData fields initialized to empty, never fetched

Interaction Spec

UX Rules
  • Enter sends in all compose inputs (Space, DM, SMS, Email). Shift+Enter inserts a line break.
  • Esc closes the panel. Keyboard listener on the slide-out root. Closes CONNECT or OMNI, returns focus to portal content.
  • Push-not-overlay on desktop. Portal content shifts right via pr-[360px] (1024-1399px) or pr-[460px] (≥1400px). Mobile uses full-screen overlay with a dark backdrop. Behavior is already correct in both panels — preserve it exactly.
  • 2-pane: left rail is always visible on desktop. Selecting a Space, DM, or conversation does NOT replace the list. On mobile (full-screen overlay), the left rail slides away and a back button appears in the thread header.
  • Log row click → jump to conversation. Clicking an SMS or Email row in the Log tab sets the active tab and pre-selects that thread in the left rail. Clicking a voice row opens an inline detail drawer within the Log tab (no tab switch).
  • View Client chip → client detail. Any thread with a known client_id shows a "View Client →" link that navigates to /clients/{clientId}. Navigating closes the slide-out.
  • Panel title contextualizes to active content. When a Space is open: title shows "#team-rpi". When nothing selected: title shows "CONNECT". Same pattern in OMNI: "Mary Johnson" when a thread is open, "OMNI" at rest.
  • Inbound call float is always mounted. InboundCallCard renders at the portal root level — outside both slide-outs — so the ring card appears over any open panel. Behavior unchanged; only the visual is rebuilt.
  • Active call = full-screen takeover. Once answered, ActiveCallScreen overlays the portal. Both slide-outs are hidden. Call end restores prior view. State machine unchanged; visual rebuilt.

Out of Scope — V1

Deferred to V2

These are explicitly deferred. None block the V1 rebuild.

✗Typing indicators in Spaces or DMs (requires WebSocket or push channel)
✗Message edit / delete in CONNECT (Google Chat API supports it, CONNECT V2)
✗Thread/reply nesting within a Space (Google Chat thread model, CONNECT V2)
✗File attachments in CONNECT (Drive file picker, CONNECT V2)
✗Read-receipt write-back to /spaces/:id/read-state (backend placeholder only)
✗SMS group messaging (multi-recipient threads, OMNI V2)
✗HTML rich-text email composer (OMNI V2 — V1 uses plaintext)
✗Meet transcript viewer (conferenceRecord linkage deferred, connect.ts:486-488)
✗Cmd+K quick-switcher across spaces and DMs (Sprint 11+ keyboard layer)
