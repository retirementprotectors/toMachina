§ 03

Google Chat + Meet API — Capability Table

🔑 Internal OAuth Unlock — The Single Biggest Force-Multiplier On This Plan

One GCP setting change. Eliminates every consent screen. Unlocks 8 Chat scopes instantly.

Our GCP OAuth client is currently set to External — meaning every @retireprotected.com user gets a consent screen and requires explicit scope grants. Flipping it to Internal (one checkbox in GCP Console, ~30 seconds, only valid if no external consumers exist) eliminates the consent screen entirely. All 8 Chat scopes below become silently granted at Firebase Auth login. Every CONNECT feature listed as "Needs OAuth unlock" ships the same sprint this flips.

§ 07

Risks & Mitigations

RiskLevelMitigation
A2P campaign still blocked (OMNI) — Twilio local SMS blocked until TKO-COMMS-001 ships High TKO-COMMS-001 is ticket #1. Opt-in landing page (MUSASHI copy + TAIKO wire) is the unblock. Toll-free +1 888 620 8587 carries inbound/outbound in the interim.
GCP OAuth set to External — consent screen blocks users on first scope grant Medium Flip to Internal (one GCP Console checkbox, ~30 seconds). Only valid if no external OAuth consumers exist — verify first. Unlocks 6 Chat scopes instantly.
Pub/Sub infra complexity — real-time messaging via Pub/Sub is non-trivial Medium Polling MVP ships first (5-sec GET on messages.list). Pub/Sub is v2 (TKO-CONN-008) — users get real-time feel without Pub/Sub complexity at launch.
No native Huddle equivalent — Slack Huddles are 1-click, Meet requires a meeting object Low Meet 1-click button embedded in every Space header and DM thread. Meet API v2 creates the space in under 500ms — effectively 1-click from the UX perspective.
OAuth scope consent (one-time) — even Internal OAuth shows a consent screen at first login Low One-time only at Firebase Auth login. Transparent to users after first session. Documented in onboarding flow (MUSASHI handles copy).
Historical message migration — Slack history cannot cleanly import to Google Chat Low Not migrating. Slack workspace stays accessible (read-only after cutover). Fresh start in CONNECT. JDM confirmed: "DGAF about history."
§ 08

Sprint Plan — Two Parallel Tracks

OMNI unblocks the broken pipe first (A2P). CONNECT builds the new one. Both run simultaneously — no dependency between them.

📡 OMNI Track — TKO-COMMS-*
TKO-COMMS-001
A2P Opt-In Landing Page + Twilio Campaign Unblock

Build TCPA-compliant opt-in page (MUSASHI writes copy). Submit Twilio A2P campaign. Reopen ticket 25630009. Highest-priority ticket on the entire platform.

TKO-COMMS-002
Push-Padding Wiring — Notification Routing

Wire the notifications collection to actual channels. Add channel selection logic (SMS vs email vs in-portal toast) based on event type and user preference.

TKO-COMMS-003
InboundCallCard Fix — Broken Inbound UX

Diagnose and fix the inbound call UI card. Call arrives, card should appear with client context loaded. Currently broken — agent picks up in the dark.

TKO-COMMS-004
Inbound Email Webhook — SendGrid → Firestore

Wire SendGrid inbound parse webhook to the communications collection. Replies from clients land in OMNI Log automatically.

TKO-COMMS-005
SendGrid Delivery Audit

Pull 90-day delivery stats. Identify bounce rate, open rate, failed sends. First time this has ever been audited. Establish monitoring baseline.

TKO-COMMS-006
Vendor Labels Cleanup — communications collection

Audit and normalize vendor_label field across all communications records. Remove orphaned labels from deprecated integrations (GHL, old RAPID_COMMS).

🏠 CONNECT Track — TKO-CONN-*
TKO-CONN-001
OAuth Scope Extension — Flip to Internal

Extend Firebase Auth consent to include 8 Google Chat scopes + 2 Meet scopes. Flip GCP OAuth to Internal. Gate: all team members auth without consent screen.

TKO-CONN-002
Chat Spaces List + Read — API Route

Build GET /api/connect/spaces and GET /api/connect/spaces/:id/messages. Returns Space list + message thread. Depends on TKO-CONN-001.

TKO-CONN-003
ConnectPanel — Kill Mocks, Wire to API

Remove all seeded/mock data from ConnectPanel. Wire Channels tab to /api/connect/spaces. First live messages appear inside ProDashX.

TKO-CONN-004
DM List + Read + Write

Build DM Space list API. Wire DMs tab. Implement compose and send. Gate: team members can DM each other from inside ProDashX.

TKO-CONN-005
Meet API v2 — Transcripts + Recordings

Extend meet_create_meeting to fetch transcript and recording URLs post-call. Surface in Meet tab history. Gemini AI notes if transcript is available.

TKO-CONN-006
Read State + Unread Badges

Implement chat.users.readstate scope. Track read position per Space per user. Render unread badge counts in sidebar. Feels like a real chat application.

TKO-CONN-007
Polling Real-Time — 5-sec Refresh MVP

Wire 5-second polling on the active conversation. New messages appear without page refresh. Not Pub/Sub — fast enough for MVP, no infra complexity.

TKO-CONN-008
Pub/Sub Real-Time — v2 Follow-Up

v2 — not in first sprint. Replace polling with Google Cloud Pub/Sub push webhook → SSE stream to portal. True real-time. Runs after polling MVP is battle-tested in production.
