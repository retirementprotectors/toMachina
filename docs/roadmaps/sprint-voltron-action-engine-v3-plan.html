Architecture Constraints

  1. ATLAS Mirror Pattern — VOLTRON mirrors ATLAS layered architecture: Atomic → Super Tool → Wire
  2. Auto-Generated Registry — 4-source parser (API routes, Firestore schemas, MCP manifests, Voltron tools) → unified voltron_registry
  3. Wire Executor — Sequential execution with output chaining, approval gates, SSE progress, Firestore audit trail
  4. Entitlement Filtering — 5 roles: COORDINATOR(1), SPECIALIST(2), DIRECTOR(3), VP(4), ADMIN(5)
  5. MDJ_SERVER owns Agent SDK — /home/jdm/mdj-agent hosts all claude-agent-sdk code. Cloud Run (tm-api) is REST-only
  6. TypeScript Strict — No any types. All exports typed. npm run build must pass
  7. Firestore Collections — voltron_registry, wire_executions, users, clients/{id}/activity_log
  8. SSE Streaming — 6 event types: stage_start, stage_complete, stage_error, approval_required, wire_complete, wire_error
  9. Model IDs — claude-opus-4-6, claude-sonnet-4-6, claude-haiku-4-5-20251001
  10. Idempotent Registry — Re-runs of npm run generate:voltron-registry produce identical output
