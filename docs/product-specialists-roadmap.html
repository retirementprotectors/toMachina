The 4-Phase LTC Framework (RPI Proprietary)

Phase I: Self-funded (savings + income)
Phase II: Annuity-funded (LTC riders, chronic illness benefits)
Phase III: Insurance-funded (standalone LTC, hybrid policies)
Phase IV: Government-funded (Medicaid, VA benefits)

Gaps

  • No standalone legacy/estate planning beyond LTC-through-annuity
  • No elder law/probate integration
  • No EHR integration for health-triggered LTC qualification
  • No Medicaid spend-down calculation

Specialist Architecture

How It Works

Agent says: "Run tax harvesting on the Nepper account"
    │
    ▼
VOLTRON detects INVESTMENT domain
    │
    ▼
Switches to INVESTMENT SPECIALIST mode
    │
    ▼
Chains: get_client → get_accounts → analyzeTaxHarvest → generateCasework
    │
    ▼
Output: Branded PDF in ACF folder + Slack notification to agent

Golden Rule

Collections are the SSOT. Each specialist reads from Firestore collections — never text fields. If a dropdown exists for carriers, products, or agents, the specialist uses it. Text fields referencing collection data = WRONG.
