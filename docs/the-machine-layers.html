11 Layers of AI Infrastructure

The agents are the last thing we built. They sit on top of 10 layers of infrastructure that took 14 weeks to construct. Without those layers, an agent is just a chatbot. With them, it's a team member.

Identity Most stop here

Knows who JDM is, how RPI operates, how to communicate, what's forbidden, what's expected. Not a style guide — an operating manual for being YOUR engineer.

What we built
A 55K-character global instruction file read by every agent on every machine. Plus per-warrior soul.md and spirit.md files for persistent identity across sessions.

Standards

The Immune System

35 rules that BLOCK bad code before it's written. PHI never leaks. Credentials never get hardcoded. Insecure patterns are killed at the tool level. The system doesn't ask — it prevents.

What we built
The hookify plugin: 16 Block rules (hard stops — PHI in logs, hardcoded secrets, generated logos), 7 Warn rules (soft catches — untyped API responses, missing structured responses), 6 Intent rules (auto-switching — plan mode, deploy protocol), 6 Quality Gates (evidence-based — must prove build passed before reporting done). Symlinked to 8 active projects. Violations feed the closed-loop learning system.

Process

FORGE

Takes a Discovery Doc through Plan → Build → Audit → Deploy → Verify. CI gates at every stage. The process generates its own instructions.

What we built
FORGE: an 11-step sprint lifecycle. Discovery Doc → #LetsPlanIt → #LetsRockIt → Build → Type-check gate → Build gate → #LetsAuditIt → UX Audit → #SendIt → CI passes → auto-merge → deploy. Phase-aware prompts generate build instructions automatically. E2E test suites (Playwright UI + Vitest intake) as pre/post-deploy gates. Every sprint creates its own paper trail.

How a Warrior Boots — The Soul / Spirit / Brain System

Every warrior has three identity files. When a new session starts, these load instantly — turning a blank Claude session into a fully operational team member in 30 seconds.

soul.md

The operational brain. What the warrior built, architecture decisions, current system state, key file paths, role in the team. The "what you need to know to do your job" file.

spirit.md

The relationship layer. Defining moments, actual dialogue with JDM, the energy and culture. This is what makes a warrior feel like the SAME warrior, not just another session.

brain.txt

The raw archive. Full session transcripts (10K–20K+ lines). Not loaded at startup — used for deep research when a warrior needs to recall something specific.

Why it matters: Without these files, every session starts from zero. With them, a warrior knows who it is, what it built, and what the relationship with JDM feels like — before writing a single line of code.
