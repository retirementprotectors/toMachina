What exists, what’s missing, and what to build

VOLTRON's QUE registry is already calculation-complete for JDM's casework spreadsheets. Every formula in ANNUITY- GROWTH MAX, Income Comparison, VA DB Illustration, and Index + MVA Calcs maps cleanly to an existing calc-* tool. The super-tools ANALYZE_GROWTH, ANALYZE_INCOME_LATER, and ANALYZE_MGE compose those atomic calcs into the end-to-end pipelines the spreadsheets describe.

The gaps are presentation (HTML renderers that mirror JDM's spreadsheet visual), multi-product side-by-side (current super-tools are 2-way CURRENT vs PROPOSED; the FIA Head2Head needs N-way), and educational concept generators (Asset Types, Balanced Portfolio, Accumulation vs Distribution — these are visual aids, not calcs).

The Opportunity Folder Builder is an ACF stage tracker, not QUE — belongs in MEGAZORD's registry.

How VOLTRON should use this tonight:
  1. For any client projection (Ruth / Westfall / Thurman), use ANALYZE_GROWTH or ANALYZE_INCOME_LATER — do NOT hand-calculate year-by-year depletion.
  2. For multi-carrier side-by-side (Mitchell diligence), note the gap — N-way comparison isn't a super-tool yet. Call calc-fia-projection 3x with different product inputs and assemble manually, or stage the build-out.
  3. For the 4-Phase LTC framework (Ruth's COF LTC policy): ANALYZE_LTC exists.
  4. For Roth conversion math (Ruth's $39K × 3yr plan): ANALYZE_ROTH exists.
  5. Presentation — the spreadsheets have a specific visual (Client Name header / Contract Basics grid / Year-by-year projection table). generate-summary-html and generate-detail-html exist but don't currently mirror this layout. Tracked as Build Plan item 5.

Build-out Plan — DOCUMENT ONLY (no implementation tonight)

Per JDM’s directive: sketch the super-tools + renderers to mirror the 13 xlsx components. Sequence from lowest-cost / highest-leverage to stretch.

1. calc-accumulation-depletion (unified API)

What: Canonical name for the (Start − W/D) × (1 + NetReturn) formula. Aliases calc-va-depletion.

Inputs: { startingValue, annualWithdrawal, grossReturn, totalFeeRate, years } (identical to calc-va-depletion).

Output: { schedule: YearRow[], depletionYear: number | null } — same shape as existing.

Cost: ~30 min. Add file, add registry entry, add alias export in tools/index.ts.

2. analyze-accumulation-projection (composition)

What: Super-tool composing calc-accumulation-depletion + calc-household-aggregate. Takes a household + 1 product and returns a complete projection with client demographics baked in (primary age, joint age, policy year all derived from DOB + issue date like the spreadsheet does).

Composes: calc-household-aggregate, calc-accumulation-depletion, calc-fia-projection (conditional on product type).

Output shape:

{ household: HouseholdSummary, currentProjection: DepletionSchedule, product: ProductSpec, ageTimeline: { primaryAge: number[], jointAge: number[], policyYear: number[] } }

UI surface: Reuses generate-summary-html + (new) generate-spreadsheet-mirror-html.

Cost: ~90 min.

3. analyze-accumulation-comparison (N-way side-by-side)

What: Multi-product comparison matching FIA Head2Head layout. Accepts an array of products + a household, returns ranked comparison.

Composes: analyze-accumulation-projection × N, then calc-delta against the first as baseline.

Output shape:

{ baseline: ProductProjection, alternatives: ProductProjection[], ranked: { productId, totalReturn, annualizedPct, accountValue, surrenderValue, netAdvantageVsBaseline }[], sideBySide: ComparisonRow[] // year 1..N with one column per product }

Cost: ~2 hr.

4. Upgrade analyze-income-later to match Income Comparison Master template

What: Current super-tool returns rollup + GMIB calcs. Upgrade to emit the Income Comparison Master visual structure: CURRENT Benefit Base table (rows = policy years, cols = BenefitBase / GMIB / Bonus% / Bonus$ / GMIB% / GMIB$) stacked above PROPOSED rollup table with same column layout.

No new calcs. Pure presentation upgrade. Update output shape to return { current: IncomeComparisonRows, proposed: IncomeComparisonRows, delta: DeltaRow[] } instead of flat rollup schedule.

Cost: ~60 min (no new calc tools needed).

5. generate-spreadsheet-mirror-html (new generator)

What: HTML renderer whose layout mirrors JDM’s spreadsheet visual — Client Name + Account # header row, Contract Basics grid (Issue Date / AV / SV / Deposit-W/D / As-Of / Annualized% / M+E+A), GMIB details block, GMDB details block, year-by-year projection table with CURRENT above PROPOSED.

Templates needed:

  • growth-max.template.html — for ANALYZE_GROWTH output
  • income-later.template.html — for ANALYZE_INCOME_LATER output
  • head2head.template.html — for ANALYZE_ACCUMULATION_COMPARISON output (N columns)

Print-CSS ready (per HTML-is-the-brochure-production-path rule) — Poppins + Mulish + RPI navy, page-break-inside:avoid on year-row groups.

Cost: ~3 hr for all three templates.

6. calc-portfolio-income-matrix + ANALYZE_INCOME_PORTFOLIO (stretch)

What: Atomic calc that aggregates accounts by custodian × asset class × income type (matching Income Portfolio Analysis sheet). Super-tool wraps it with advisory fee calcs and income totals.

Cost: ~90 min.
