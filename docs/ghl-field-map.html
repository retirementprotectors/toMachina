Architecture Decisions (CIO scope)

  • Rider details → JSON sub-documents (riders[] array), NOT 45 flat fields
  • Surrender schedule → surrender_schedule: number[] array (15 values), NOT 15 separate fields
  • Vesting schedule → vesting_schedule: number[] array (11 values), NOT 11 separate fields
  • Payout bands → riders[].payout_bands[] array of {start_age, end_age, payout_factor}
  • Beneficiary summary on account (name + %), full detail in Beni Center module
  • CashFlow stays at account level (shared between Annuity + Investment)
  • RMD → RMD Center module, NOT on account
  • Decedent info → nested object or flat fields with decedent_ prefix (estate accounts only)
  • Advisory Fees → Investment-only fields (3 fields: $, %, calculation method)
