#1 — Opportunity Create Error

Bug Fixes — Opportunity Lifecycle

Fix 1: Create Error (#1)

Error thrown on opportunity create

Investigate the create flow: POST /api/opportunities. Check required fields, pipeline stage lookup, and error response. May be a missing default stage or validation failure that surfaces as a generic error.

Fix 2: Duplicate Creation (#2)

Opportunity saved despite error

The API call succeeds (writes to Firestore) but the frontend displays an error anyway. Likely a response parsing issue — the create succeeds, but something in the response handling triggers the error UI. Fix: ensure create either fully succeeds (no error) or fully fails (no write).

Fix 3: Advance Stage Error (#5)

Stage transition fails on click

Investigate PATCH /api/opportunities/:id/advance or equivalent. Check stage ordering, permissions, and whether the next stage exists in the pipeline definition.

Feature: Assignee vs Agent Separation (#6)

Currently opportunities have one person field. Need two:

Agent (Writing Rep)

The licensed agent who wrote the policy. Populated from writing agents list. Already exists — this is the current "agent" field.

Assignee (Admin Staff)

The team member processing the opportunity. New field. Dropdown: Nikki Gray, Angelique Bonilla, Susan Kaelin. Drives task routing and accountability.

Feature: ACF Subfolder Display (#8)

Current: ProDashX ACF section shows a flat list of files. Actual: Google Drive has nested subfolders following the 5-folder lifecycle structure.

Current ProDashX: Actual Google Drive: ┌─────────────────┐ ┌─────────────────────┐ │ file1.pdf │ │ 📁 Applications │ │ file2.pdf │ │ ├── app1.pdf │ │ file3.pdf │ │ └── app2.pdf │ │ file4.pdf │ │ 📁 Correspondence │ │ (flat list) │ │ └── letter1.pdf │ └─────────────────┘ │ 📁 Policy Documents │ │ ├── policy.pdf │ │ └── amendment.pdf │ └─────────────────────┘

Fix: ACF component needs to render Drive folder hierarchy with expand/collapse on subfolders.

Feature: Call Notes in Client Profile (#9)

Call notes go to communications collection today. Need a dedicated "Notes" section on the client profile that surfaces call notes, meeting notes, and freeform notes without opening the COMMS slide-out.

Infrastructure Exists

clients/{id}/activities subcollection already exists. Call notes can be surfaced by filtering activities with type: 'call_note' and rendering them in a new tab/section on the client profile.

Feature: Phone/Dialer Minimize (#10)

Active call locks the UI. Need a minimize/background mode — click to collapse dialer to a floating mini-bar, navigate client profile freely, expand back when needed.
