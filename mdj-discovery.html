The Problem

Josh Millang is the single point of capability at RPI. He can answer any question, execute any action, access any system — in 5 seconds. But there's only one Josh. When he's on a call, everyone waits. When he's asleep, everything waits.

The Solution

MyDigitalJosh (MDJ) — an AI assistant wired into the ENTIRE toMachina platform that gives every team member Josh-level capability, 24/7, simultaneously. Not a chatbot. Not a search tool. A digital clone of Josh's ACCESS and CAPABILITY, scoped by role.

The Outcome

Any RPI team member can ask MDJ anything about any client, any account, any process — and get an answer or action in seconds. New hires are instantly dangerous. The CEO bottleneck disappears.

Example: "What's the status of Mrs. Johnson's 1035 exchange?"

Nikki is on Mrs. Johnson's client detail page in ProDashX. She opens the MDJ panel and asks:

1

Widget Captures Context

MDJPanel detects she's on /contacts/abc123 and automatically includes client_id: "abc123" with her message.

2

API Authenticates

Cloud Run verifies Nikki's Firebase token. Loads her profile: Level 2 (LEADER), Service division. Builds permission context.

3

Agent Loads Context

MDJ1 loads conversation history + Mrs. Johnson's client insights from Firestore. Specialist router: "1035 exchange" → stays with General MDJ.

4

Claude Decides What Tools to Call

tm_get_client { id: "abc123" } and tm_get_client_pipelines { client_id: "abc123" } — both allowed for Level 2, no approval needed.

5

Tools Execute

Calls tm-api: GET /api/clients/abc123 and GET /api/flow?client_id=abc123&pipeline=nbx. Results returned to Claude.

6

Streamed Response

"Mrs. Johnson's 1035 exchange is in the New Business pipeline, stage 3 of 5 (Carrier Review). Submitted to Athene on March 18th. Transfer amount: $247,000 from Jackson National FIA. Expected completion: 5-7 business days."

7

Nikki Has Her Answer

3 seconds. No clicking. No navigation. No waiting for Josh.

Permission Model

MDJ respects the same entitlement system that gates portal access. The bot can only see and do what the user is allowed to see and do.

Level 0: OWNER (Josh + John)

Everything. All tools. All data. Deploy. Configure. No review gates. Full trust.

Level 1: EXECUTIVE

All data read/write. Can trigger imports, wires. Can access MDJ-Ops specialist. Review gate on: deploy, config changes.

Level 2: LEADER (Division Heads)

Division-scoped data. Can send comms. Can access MDJ-Securities, MDJ-DAVID. Review gate on: external emails, bulk operations.

Level 3: USER (Sales/Service Team)

Read client/account data. Run quotes. Send comms WITH REVIEW GATE. Cannot access financial data or run imports.

Review Gates

Some actions require the user to approve before executing. The bot shows the proposed action and waits for confirmation.

LevelActions Requiring Approval
Level 3 (USER)send_email, send_sms, submit_case, update_pipeline
Level 2 (LEADER)send_email (external only), bulk operations
Level 1 (EXECUTIVE)deploy, config changes
Level 0 (OWNER)None — full trust

Memory System — The Bot Never Forgets

Four tiers of memory ensure MDJ always has the right context.

Short-Term: Conversation History

Last N messages in current chat. Loaded into Claude context on every request. Survives browser refresh (Firestore-backed). Budget: ~80K tokens.

Medium-Term: Conversation Summaries

AI-generated summary when conversation exceeds 20 messages. Last 3 summaries loaded into new conversations. "Last time you discussed Mrs. Johnson, you talked about her 1035 exchange status..."

Long-Term: Client Insights

Extracted automatically from every conversation that references a client. "Mrs. Johnson prefers morning calls." "Husband handles finances when she travels." "Anxious about market volatility." Shared across all team members. Gets smarter with every interaction.

Platform: Live Data (Always Current)

The bot doesn't "remember" account values — it QUERIES Firestore in real time. Always accurate. Clients, accounts, pipelines, comms, documents. The platform IS the memory.
